## Context

ContextForge 截至 Phase 21（retrieval-quality, Done / v0.14.0）已把 reranker 推进到**trait 抽象 + 本地 cross-encoder 成立、但仅 opt-in builder seam**：`core/src/rerank/traits.rs` 的 `Reranker` trait（`rerank(&self, query: &str, candidates: &[SearchResult]) -> Result<Vec<SearchResult>, RerankError>` + `name()`；`RerankError` `ModelLoad` / `Backend` / `Other`，`#[non_exhaustive]`）；`core/src/rerank/identity.rs` 的 `IdentityReranker`（按 `score` desc + `chunk_id` asc 决定性 tie-break，`IDENTITY_RERANK_REASON` provenance marker）；`core/src/rerank/cross_encoder.rs` 的 `CrossEncoderReranker`（feature `reranker-fastembed`，本地 model，by `rr.index` 映射回 candidate、`c.score = rr.score`、provenance 注释）。但**两处诚实缺口**：

**当前 in-repo reranker 的真实状态（本 phase 要关闭的诚实缺口）**：

- **`RemoteRerankerProvider` 不存在、`select_reranker` 工厂不存在**：reranker 至今只有本地路径（`IdentityReranker` / feature-gated `CrossEncoderReranker`），**没有** remote cross-encoder over HTTP provider，也**没有**像 `select_provider`（`core/src/embedding/factory.rs:27`）那样的 reranker 选型工厂。这与 embedding 侧不同——embedding 的 `RemoteEmbeddingProvider` 自 Phase 22 已全实现（ADR-042 仅复用 + 建 harness），而 remote reranker 须**首次构建** provider + 建工厂 + 契约测试 + live harness。
- **reranker 从未在生产数据面接线**：`core/src/server.rs`（hybrid 路径 `:334` + semantic 路径 `:376` 两处）与 `core/src/server/data_plane/search.rs`（semantic 路径 `:282` 一处）只调 `.with_embedder()` + `.with_vector_searcher()`、**从不调 `.with_reranker()`**——`with_reranker` 仅是 Phase 21 引入的 opt-in builder seam、**仅测试用**。即生产路径至今无 rerank step，reranker 能力从未由 config 在生产路径生效。
- in-repo 唯一的 rerank「质量数」证据是 Phase 21 dogfood eval（本地 cross-encoder over-baseline uplift，ADR-026），**非真实 remote 模型的 rerank 质量**，且未在生产数据面接线。

**DE-RISK 已由 lead 真实证实（不是推测——真实端到端 round-trip 已跑通）**：SiliconFlow rerank 端点（`https://api.siliconflow.cn/v1/rerank`，与 embedding 同 URL 同 key、仅 model 不同）+ 模型 `Qwen/Qwen3-VL-Reranker-8B` 端到端 round-trip 真实跑通——请求 `{model, query, documents: [字符串], top_n, return_documents: false}`、响应 `{results: [{index, relevance_score}], meta}`。探针：查询「如何保存配置到文件」对 4 文档 rerank，相关文档 `config_save` `relevance_score=0.7356` 排 **#1**、近义干扰 `config_load=0.0158` 排 #2、无关项 ~`0.0006` / ~`0.0003`，HTTP 200——证明端点可用 + 排序语义正确 + 约 **46x** 区分度（0.7356 / 0.0158）。这证明 remote rerank 读写路径 + cross-encoder 语义在真实付费端点上正确，本 phase 的剩余工作是把它系统化为 RemoteRerankerProvider + select_reranker + 可复现 rerank quality harness + config env-bridge + 数据面 opt-in 接线，而非验证可行性本身。

本 Phase 38 是承 Phase 21（reranker trait + cross-encoder）+ Phase 22/37（remote provider 范式）血脉的**深化与真实兑现**——把 ADR-026 / ADR-042 honest-deferred 的 remote reranker live 维度**真实兑现**：(1) 首次构建 `RemoteRerankerProvider`（镜像 `RemoteEmbeddingProvider` 与 `CrossEncoderReranker` 结构）+ `select_reranker` 工厂（镜像 `select_provider`）+ 纯函数契约测试；(2) 建 env-gated live rerank quality harness，在作者手工标注 query×candidate 集上测 real remote cross-encoder vs IdentityReranker（no-semantic）基线的 MRR / recall@1；(3) 对一个 REAL 远程端点跑（de-risk 已证）；(4) 首次把 reranker 从 Go `[reranker]` config 经 env-bridge + Rust 数据面 `reranker_from_env` opt-in `with_reranker` 在生产路径接线。全程守 ADR-013：live rerank 质量真实数字一律**真实跑出后据实记录**（task-38.1 §10 引用主 agent 本机真实 SiliconFlow run），floor 是 guard、real-vs-baseline delta 是报告，**小型手工标注集不夸大为大基准质量断言**；tag/digest/tlog 标 `待回填` 直至发版后回填（ADR-013 不预填）。默认构建不变：`reranker-remote` opt-in/off，默认数据面 `CONTEXTFORGE_RERANKER_PROVIDER` unset → **字节等价无 rerank（向后兼容，当前行为不变）**（ADR-004/008）；**0 新依赖**（`ureq` 自 task-22.3 起已是 optional dep，新 `reranker-remote` feature 复用之）。**API KEY 永不进 config.toml**——仅由用户设 env（PRD 安全基线）。

## Decision

remote reranker live 兑现采用 **「首建 RemoteRerankerProvider + select_reranker 工厂（wire format 映射回 index、IdentityReranker no-semantic 基线对照、作者手工标注 query×candidate 集、env-gated 无密钥诚实退出）+ 真实实测 rerank 质量数（real-vs-baseline delta，据本机真实 run 据实记录）+ reranker-config-bridge-and-data-plane-wiring（Go `[reranker]` add-only + setRerankerEnv env-bridge + Rust 数据面 reranker_from_env opt-in with_reranker 三处接线，env-wins，API key env-only）+ 默认 0-network / 0 新 dep / 既有契约不变」** 策略，分 4 个决策点：

### D1 — RemoteRerankerProvider + select_reranker 工厂 + live rerank harness 方法论（real remote cross-encoder vs IdentityReranker no-semantic 基线对照；作者手工标注 query×candidate 集；env-gated 无密钥诚实退出）（task-38.1）

首次构建 `RemoteRerankerProvider`（实现 `Reranker` trait，feature `reranker-remote`）+ `select_reranker` 工厂（镜像 `select_provider`），并新增 `core/tests/remote_rerank_recall.rs`，`#![cfg(feature = "reranker-remote")]`，env-gated on `CONTEXTFORGE_RERANKER_API_KEY`（factory 另读 `CONTEXTFORGE_RERANKER_ENDPOINT` / `_MODEL` / `_PROVIDER`；api_key 永不记录）。当 `CONTEXTFORGE_RERANKER_API_KEY` 未设 → `eprintln` 一条 skip notice + `return`（honest-defer——CI 无密钥时干净跳过、**不** fail，ADR-013）。方法论：

- **RemoteRerankerProvider（wire format 纯函数 + ureq POST，镜像 `RemoteEmbeddingProvider` + `CrossEncoderReranker`）**：`build_rerank_request_body`（query + `candidates` 的 doc 文本 → `{model, query, documents: [字符串], top_n, return_documents: false}`）与 `parse_rerank_response`（`{results: [{index, relevance_score}], meta}` → 按 `index` 映射回 `candidates[index]`、set `score = relevance_score`、annotate provenance reason，镜像 `CrossEncoderReranker` 的 by-`rr.index` 映射）为**纯函数**；`ureq` POST 经 feature `reranker-remote`；`Debug` 不打印 `api_key`（安全红线）。
- **select_reranker 工厂（镜像 `select_provider`，`factory.rs:27`）**：从 config reranker 名 + env 选型——`""` / `"none"` → 无 rerank（不构建 reranker）、`"identity"` → `IdentityReranker`、`"remote"` → feature-gated `RemoteRerankerProvider`（从 env 读 `CONTEXTFORGE_RERANKER_ENDPOINT` / `_MODEL` / `_PROVIDER` / `_API_KEY`，镜像 `factory.rs:49-74` remote 分支）；feature-off 时 `"remote"` 分支**显式 `Err` 不静默**（镜像 `select_provider` 的 feature-off 显式错误）。
- **作者手工标注 query×candidate 集**：作者手工标注每 query 一个已知相关文档 + 近义干扰（镜像 embedding harness 的 `docs()` / `cases()` 风格），故意埋近义干扰项使 top-1 命中**非词面 trivial**——real cross-encoder 须用 query×doc 语义把相关文档排到 #1。
- **IdentityReranker no-semantic 基线对照**：基线 = `IdentityReranker`——候选喂入时携带统一 / 无相关性先验 score，故 `IdentityReranker` 的决定性 tie-break（`score` desc + `chunk_id` asc）≈ chance（无语义信号）；real remote cross-encoder 须用 query×doc 语义把相关文档排到 #1，两者 delta 即真实 cross-encoder 实际买到的 rerank 质量。
- **质量测量**：对每个 query 取 candidates，real reranker rerank 后判 `recall@1`（标注相关文档排第 1）+ 算 `MRR`（标注相关文档 reciprocal rank 均值）；先 `eprintln` 出 **ACTUAL 实测数**（含每 query top 排序 + relevant rank + HIT@1/MISS）再 `assert`：floor `MRR_remote >= 0.70`（防回退 guard）且 `MRR_remote > MRR_identity`（real cross-encoder 须跑赢 no-semantic 基线，否则真实模型买不到东西）。floor 是 guard、real-vs-baseline delta 是报告（真实值据本机真实 run 据实记录，ADR-013 不伪造）。
- **非网络契约 + well-formed 守护测试**：另有非网络守护测试（`build_rerank_request_body` / `parse_rerank_response` fixture 契约 + `select_reranker` 路由 + 标注集 well-formed：doc id 唯一 / 每 query relevant id 存在 / case 数门槛），**无密钥也总是跑**——使 harness 在无密钥环境也有确定性逻辑校验。

**0 新依赖（`ureq` 已 optional，新 `reranker-remote` feature 复用之）、0 schema migration、0 proto 改动、0 默认构建变更**。Tests：TEST-38.1.1（live rerank harness，env-gated `CONTEXTFORGE_RERANKER_API_KEY`——对 live remote 端点实测 `MRR_remote >= floor` 且 `MRR_remote > MRR_identity`、`recall@1`）+ TEST-38.1.2（非网络契约 + well-formed 守护——**无密钥** 也跑，断言 `build_rerank_request_body` / `parse_rerank_response` fixture 契约 + `select_reranker` 路由 + 标注集 doc id 唯一 / relevant id 存在 / case 数门槛）+ TEST-38.1.3（= LAST，D2 lint）。

**理由**：real-vs-IdentityReranker（no-semantic）基线对照是干净的 rerank 质量指标——候选携带统一 / 无先验 score 喂入时 `IdentityReranker` 决定性 tie-break ≈ chance（无语义），real cross-encoder 应用 query×doc 语义把标注相关文档排到 #1，两者 delta 即真实 cross-encoder 实际买到的 rerank 质量。作者手工标注 query×candidate 集故意埋近义干扰，使命中非词面 trivial；RemoteRerankerProvider 镜像 `RemoteEmbeddingProvider`（纯函数 wire format + ureq POST + Debug 不打印 api_key）与 `CrossEncoderReranker`（by-`index` 映射回 candidate + set score + provenance）两个已验证范式；select_reranker 镜像 `select_provider` 的 config 名 + env 选型 + feature-off 显式 Err；env-gated honest-defer（无密钥 → 干净 `return`，不 fail）使 harness 在无密钥环境（CI / 未配密钥的本机）干净跳过、不伪造 rerank 质量通过。floor 作 guard（防回退）、real-vs-baseline delta 作报告（据实记录真实 rerank 质量）——两者职责分离，呼应 ADR-006 recall-eval-acceptance-gate 的 gate 方法论与 ADR-041 的 floor-guard + 真实数回填先例。**诚实范围**：小型手工标注集，MRR floor 是回退 guard 而非质量上限断言；大语料 rerank 质量（NDCG / MRR 大基准）续 `[SPEC-DEFER:phase-future.reranker-large-corpus-quality]`。

### D2 — 真实实测 rerank 质量数（real-vs-baseline delta，据主 agent 本机真实 SiliconFlow run 据实记录；CI honest-defer 因 remote 是付费外部 API 无免费 service container）（task-38.1 §10）

harness（D1）跑出的真实 rerank 质量数字是本 phase 的真实 rerank 质量证据。据 ADR-013，规划阶段（Proposed）**不预填系统 MRR / recall@1**——真实数 task-38.1 实测后据实回填（不伪造、不预填，ADR-013）。当前可引用为**可行性证据**的是 lead de-risk 探针真实结果：对真实 SiliconFlow `Qwen/Qwen3-VL-Reranker-8B` 端点，查询「如何保存配置到文件」对 4 文档 rerank，相关文档 `config_save` `relevance_score=0.7356` 排 **#1**、近义干扰 `config_load=0.0158` 排 #2、无关项 ~`0.0006` / ~`0.0003`，HTTP 200——证明端点可用 + cross-encoder 排序语义正确 + 约 **46x** 区分度（0.7356 / 0.0158）。系统 MRR / recall@1 over 作者标注集的真实数将在 task-38.1 实测后记入 task-38.1 §10 + ADR-043 §Ratification + `docs/releases/v0.31.0-evidence.md`，并标注是主 agent 本机真实 SiliconFlow run（非合成、非 CI）。

**CI honest-defer（与 qdrant 的诚实差异）**：与 Phase 36 qdrant 不同——qdrant 有免费 OSS service container 可接进 CI 拓扑（ADR-041 D3 永久守护）；remote reranker 是**付费外部 API、无免费 service container**（与 remote embedding 同理），故 CI 无密钥时 harness 干净 honest-defer skip，rerank 质量由**本机已认证 run 实测**而非每次 CI run 守护，复用既有 `[SPEC-DEFER:phase-future.embedding-remote-ci-credential]`（reranker 与 embedding 是同类付费 API 凭据约束，**不另造 reranker-ci-credential**）。这是 remote 与 qdrant 的结构性差异，据 ADR-013 据实记录、不夸大为「CI 每次 run 守护」。

**理由**：ADR-013 禁伪造红线——rerank 质量数必须真实跑出后据实记录，不伪造合成数、规划阶段不预填。本 phase 的 de-risk（real SiliconFlow `Qwen3-VL-Reranker-8B` round-trip 已证 `config_save` 0.7356 排 #1 vs `config_load` 0.0158、HTTP 200、约 46x 区分度）证明可行性 + cross-encoder 排序语义正确；系统 MRR / recall@1 over 标注集待 task-38.1 实测后据实回填——floor（`MRR >= 0.70`）是 guard、real-vs-baseline delta 是报告。**诚实判读**：小型手工标注集，de-risk 探针证明模型把明显语义对排在近义干扰之上、**非大基准质量断言**；大语料 rerank 质量续 `[SPEC-DEFER:phase-future.reranker-large-corpus-quality]`。

### D3 — reranker-config-bridge-and-data-plane-wiring（Go `[reranker]` add-only + setRerankerEnv env-bridge + Rust 数据面 reranker_from_env opt-in with_reranker 三处接线；env-wins；API key env-only 永不进 config；默认 unset 字节等价无 rerank 向后兼容）（task-38.2）

首次把 reranker 从 config 在生产数据面 opt-in 接线（reranker 至今仅 Phase 21 opt-in builder seam、仅测试用，从未由 config 在 `server.rs` / `data_plane/search.rs` 生产路径生效）：

- **Go `[reranker]` 段 add-only**：`internal/config/config.go` 新增 `RerankerConfig`（镜像 `RemoteProviderConfig`，`config.go:52`），含 `Enabled` / `Provider` / `Endpoint` / `Model`（toml round-trip 保真，`assignReranker` 镜像 `assignRemote`（`config.go:321`）+ toml serialize / decode 同步，镜像 `config.go:204` / `:238` / `:268`）。**无 `api_key` 字段**（安全红线）。
- **新 `setRerankerEnv` helper**：镜像 Phase 37 的 `setRemoteEnv` / Phase 34 的 `setVectorEnv`（`cmd/contextforge/main.go:340-385` / `:288`）跨进程 env-bridge——当 `[reranker]` 段存在且对应 env 未设时，导出 `CONTEXTFORGE_RERANKER_ENDPOINT` / `_MODEL` / `_PROVIDER` 给 spawned core daemon。**ENV WINS**（显式 env 覆盖 config 文件，向后兼容）；无 `[reranker]` 段 → 不导出 → 默认行为不变；**`api_key` 不处理**（安全红线）。`setRerankerEnv` 接线进 doServe（`main.go:108-112`）+ doMCP（`main.go:146-150`）（镜像 setRemoteEnv 接线）。
- **Rust 数据面 opt-in 接线（NEW，首次生产路径消费 reranker）**：新增 `reranker_from_env()` 读 `CONTEXTFORGE_RERANKER_PROVIDER` → 非空 / 非 `"none"` 时 `select_reranker` → `with_reranker`；接进三处生产路径——`core/src/server.rs` hybrid 路径（`:334`）+ semantic 路径（`:376`）+ `core/src/server/data_plane/search.rs` semantic 路径（`:282`）（现仅 `.with_embedder().with_vector_searcher()`、从不 `.with_reranker()`）。**默认 `CONTEXTFORGE_RERANKER_PROVIDER` unset → 不构建 reranker → 字节等价无 rerank（向后兼容，当前行为不变）**；set `"identity"` → `IdentityReranker` 生效（结果带 `IDENTITY_RERANK_REASON` provenance marker）。
- **安全红线**：API KEY 永不进 `config.toml`——`RerankerConfig` 无 `api_key` 字段、`setRerankerEnv` 不处理密钥；`CONTEXTFORGE_RERANKER_API_KEY` 仅由用户设 env（PRD 安全基线）。

## Alternatives

- **A1（纯函数契约测试充当 rerank 质量背书）**：用 `build_rerank_request_body` / `parse_rerank_response` fixture 契约测试充当 remote rerank 质量背书。否决：纯函数契约测试证明请求/响应序列化正确、`select_reranker` 路由正确，**不是真实 rerank 质量测量**（违 ADR-013 禁伪造红线）——它不触网、不验证真实 cross-encoder 的语义排序；据 D1/D2，真实 rerank 质量须由 harness 对 live 远程端点真实跑出后据实记录，floor 是 guard、real-vs-baseline delta 是 evidence。
- **A2（合成 / 伪造 rerank 分数充当 real 质量）**：用本地合成 relevance_score 或伪造的远程响应充当 real rerank 质量。否决：合成 / 伪造分数**无真实语义**——IdentityReranker no-semantic 基线已是对照项（候选携统一先验 score → tie-break ≈ chance、无语义），用合成数冒充 real cross-encoder rerank 会伪造质量（违 ADR-013）；据 D2，real rerank 质量须由真实远程模型（SiliconFlow `Qwen3-VL-Reranker-8B`）真实跑出，real-vs-IdentityReranker delta 才是真实 evidence。
- **A3（remote reranker 接 CI service-container 守护，镜像 qdrant）**：把 remote reranker 接进 CI service-container 使质量每次 CI run 守护（镜像 ADR-041 D3 qdrant）。延后 `[SPEC-DEFER:phase-future.embedding-remote-ci-credential]`（复用 embedding 的，不另造 reranker-ci-credential）：remote reranker 是**付费外部 API、无免费 OSS service container**（与 qdrant 的 `qdrant/qdrant` 免费镜像不同，与 remote embedding 同类约束）——无法像 qdrant 那样接进 CI 拓扑零成本守护；CI 无密钥时 harness 干净 honest-defer skip，质量由本机已认证 run 实测。这是 remote 与 qdrant 的结构性差异，据 ADR-013 据实分级、不强行套用 qdrant 的 CI 守护模式。
- **A4（大语料标准化 NDCG 基准）**：以标准化大语料基准（如 BEIR / MS MARCO 子集）为 ground truth 测 rerank NDCG。延后 `[SPEC-DEFER:phase-future.reranker-large-corpus-quality]`：作者手工标注 query×candidate 集（每 query 一相关文档 + 近义干扰）是 model-dependent rerank 质量的干净 primary——证明真实 cross-encoder 把明显语义对排到 #1（de-risk 探针 `config_save` 0.7356 vs `config_load` 0.0158）；大语料标准化基准引入数据集体量 + 外部依赖，非本 phase scope，作 follow-up 延后而非本 phase primary。
