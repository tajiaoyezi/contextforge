# Phase 18 spike — `hnsw`

> Backend: `instant-distance` 0.6.1 (pure-Rust HNSW approximate nearest neighbour).
> Generated by `contextforge-bench` (task-18.2 harness). **First backend with real recall data.**

Cross-check — `noop` baseline, same params (n=5000/dim=64): recall@5/10 = 0.0, P95 ≈ 0.0001ms
(empty hits) — confirms the harness measures the backend, not the scaffolding.

Debug run (n=2000/dim=64... dim=32) also gave recall@5/10 = 1.0 (recall is build-mode-independent),
P95 0.79ms, cold-start ~2.2s (debug graph build).

## Linux x86_64 re-run (task-18.7 — fills the RSS dims + the 100k scale point)

The original table above was measured on the Windows dev box, where RSS sampling is unavailable. Re-run
on Linux (WSL2 Ubuntu 26.04, release) fills the RSS dimensions and adds the n=100000 PRD-scale point:

| dimension | n=5000 | n=100000 |
|---|---|---|
| recall@5 / recall@10 | 1.0 / 1.0 | 1.0 / 1.0 |
| P95 latency (ms) | 0.382 | 0.871 |
| idle RSS (MB) | 4.38 | 55.33 |
| index RSS (MB) | 11.04 | **180.01** |
| cold-start (ms) | 836.4 | **28432.4** |
| reindex (ms) | 797.6 | 28984.0 |

The 100k point exposes hnsw's defining cost: the HNSW **graph build is ~28 s at 100k** (O(n·log n) —
up from 0.84 s at 5k) and the index footprint is **180 MB** (the heaviest in-process of the four
backends). Query latency stays sub-ms (ANN). See `docs/spikes/phase-18-comparison.md` for the full
4-backend comparison and `docs/decisions/adr-023-vector-backend-default.md` for the selection.

## Environment

- platform / arch: `x86_64-pc-windows-msvc`
- toolchain: rustc 1.95.0, release profile
- embedding source: deterministic seed vectors (unit-normalized; euclidean distance is monotonic
  with cosine for unit vectors, so HNSW nearest == the harness's cosine ground truth)

## Trade-off discussion

- **build / compile complexity**: ✅ pure Rust, **0 native/C deps** (num_cpus / ordered-float /
  parking_lot transitive). Builds cleanly on Windows MSVC *and* Linux — unlike sqlite-vec
  (`docs/spikes/phase-18-sqlite-vec.md`, MSVC build-blocked).
- **persistence model**: in-memory graph; no on-disk format yet
  ([SPEC-DEFER:phase-future.hnsw-graph-persistence]). cold-start/reindex measured via in-memory
  rebuild. A real deployment would serialize the graph or rebuild on load — vs ADR-002's
  SQLite+Tantivy persistence, HNSW needs its own persistence layer.
- **API stability / ecosystem maturity**: instant-distance 0.6.1, small focused crate; pinned in
  Cargo.lock. No incremental insert (whole-graph build) — maps to task-18.1 full-reindex semantics.
- **verdict**: **viable on all P0 platforms** and meets P95 < 500ms with room to spare. recall=1.0
  on synthetic is near-ideal (well-separated random vectors); the representative recall ranking vs
  other backends must come from the dogfood corpus + a Linux release run at the 100k target.
