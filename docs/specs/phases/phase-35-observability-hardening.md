## 2. 业务价值

热路径静默错误的可观测性硬化——把热路径中被静默吞掉的真实错误显式化（surface genuinely-swallowed errors），镜像仓库既有 stderr 惯例（Rust `eprintln!` / Go `fmt.Fprintf(os.Stderr)`），让 ops 在 disk-full / lock / index-desync / malformed-config 等真实故障下能从 stderr 注意到，而非沉默降级。**本 phase 的 ADR-013 核心价值是诚实收敛**：survey 初列 7 个候选点，经 grounding 据实收敛为 3-4 个真正静默的点——其余 4 个为「已显式化」或「设计内有意为之」，不改代码、记为 grounding 校正（honest over padding，ADR-013，第三轮债清理边际递减亦据实保持小而非凑数）：

### 35.1 rust-silent-failure-surfacing（rust-silent-failure-surfacing，🟢）

- **KEPT (a) `core/src/jobs/index_session_backend.rs:201`**：`let _ = store.append(&job_id_context, "indexing", evt.processed_files, evt.total_files, "")` **静默吞掉**一个真实持久化错误——`indexing_event_store` SQLite append 失败（disk full / lock）。本 phase 改为 `if let Err(e) = store.append(…) { eprintln!("WARN indexing-event persist failed (job={job_id_context}): {e}"); }`，**仍 best-effort（不阻断 indexing）—— observability only**，镜像同文件 :196-198 注释已声明的 best-effort 形态与 `search.rs:109` 的 `if let Err(e) = … { eprintln!("WARN …: {e}") }`。
- **KEPT (b) `core/src/retriever/mod.rs:415`**：`let (file_path, content, indexed_at) = match row { Ok(t) => t, Err(_) => continue }` 在 Tantivy 与 SQLite 不同步（desync）时**静默 skip 该 search hit**、吞掉真实错误。本 phase 改为 `Err(e) => { eprintln!("WARN retriever: chunk {chunk_id} present in index but missing from SQLite (desync), skipping: {e}"); continue; }`，**行为（skip）保持—— observability only**，镜像 retriever 既有 `eprintln!`（filter no-op 路径附近）惯例。

## 7. 阶段级风险

- **R1（中）Rust surfacing 误把 best-effort 改成 fail-fast / 破既有 skip 行为**：`index_session_backend.rs:201` / `retriever/mod.rs:415` 若把 `let _ = …` / `Err(_) => continue` 改成 return error 或 abort caller，会阻断 indexing / 中断 query，破 ADR-004 既有契约。
  - **缓解**：task-35.1 严格保持 best-effort——`store.append` 改为 `if let Err(e) = … { eprintln!("WARN …"); }`（无 `?`、无 return、不阻断 indexing）；`retriever` 改为 `Err(e) => { eprintln!("WARN …"); continue; }`（保 `continue` skip 语义）；镜像 `search.rs:109` 形态；TEST-35.1.1 断言 index session 仍成功完成；TEST-35.1.2 断言 desync hit 被 skip 且 query 返其余有效 hit 无错。stop-condition：若 indexing 被阻断 / query 中断 / 既有测试退化则 AC1 不标 `[x]`。
- **R2（中）Go surfacing 破 setVectorEnv env-only / env-wins / restore 契约**：`main.go:297-300` 若把 `config.Load` 错误改成 return error 而非 best-effort 显式化，会破 env-only 路径（malformed config 应退回 env-only，task-34.2 契约）。
  - **缓解**：task-35.2 仅在 `if err != nil { return restore }` 的 return 前加 `fmt.Fprintf(os.Stderr, …)`（return 行为不变、仍退回 env-only）；env-wins（:305-307 `LookupEnv` 守护）+ restore 闭包不动；镜像 `daemon/rest.go:110`；TEST-35.2.1 断言 malformed → WARN + env-only 路径不变 / valid|missing → 无 WARN / env-wins + restore 保持。stop-condition：若 env-only 路径 / env-wins / restore 退化则 AC2 不标 `[x]`。
- **R3（中）memstore nil-sink 误把设计内 fallback 当债 / 加噪音 WARN**：`memstore.go:579` nil-sink no-op 若被当作必修债、在每次 nil-sink 丢事件时 WARN，会在 fallback/test double 路径刷屏成噪音。
  - **缓解**：task-35.2 把 nil-sink 标 🟡 pending impl-grounding——实施时先 ground `MemMemoryStore` 是否生产路径接入期望 sink：IF 是 → `sync.Once` 一次性 WARN（不噪）；IF fallback/test-only by design → DROP 记诚实非问题 `[SPEC-DEFER:phase-future.memstore-degraded-observability-warn]`；**不预断为必交付**（ADR-013）。stop-condition：若在 fallback 路径加每事件 WARN 噪音则越界——收敛为一次性 WARN 或 DROP。
- **R4（低）误引入新 logging/metrics framework 破 0-dep / 简单优先**：observability 主题易诱导在 core 加 `log`/`tracing`/metrics crate 或为 `search.rs:109` 加结构化计数器。
  - **缓解**：task-35.1/35.2 仅用既有 `eprintln!`（Rust core 无 log/tracing/metrics crate，atomics 仅测试态）+ `fmt.Fprintf(os.Stderr)`（Go 无 metrics facility），severity 为消息串前缀；`search.rs:109` 计数器据 grounding DROP（core 无 metrics facility，加 = 过度工程，ADR-040 A1 REJECTED）；Rust 0-dep 保持（ADR-008）；metrics facility honest-defer `[SPEC-DEFER:phase-future.observability-metrics-facility]`。stop-condition：若引入新 logging/metrics framework 则越界。
