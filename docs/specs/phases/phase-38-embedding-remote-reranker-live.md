## 1. 阶段目标

v0.30.0 ship 后，ContextForge 的 reranker 抽象自 Phase 21（ADR-026, v0.14.0）已**全实现**——`Reranker` trait + model-free `IdentityReranker`（决定性默认）+ feature-gated 本地 `CrossEncoderReranker`（fastembed）+ `with_reranker` opt-in builder seam（`retriever/mod.rs:630`）——但有两个缺口：(a) **远程 reranker provider 从未存在**：`RemoteRerankerProvider` 不存在、`select_reranker` 工厂不存在，真实远程 reranker（cross-encoder over HTTP）从未联调、真实 rerank 质量从未跑出；(b) **reranker 从未在生产数据面接线**：`server.rs`（hybrid :346 + semantic :391）与 `data_plane/search.rs`（semantic :285-287）至今只调 `.with_embedder().with_vector_searcher()`、从不调 `.with_reranker()`，reranker 仅在 Phase 21 测试 seam 生效、生产路径 0 rerank。本 phase 关闭两缺口、兑现 ADR-026 记的 `[SPEC-DEFER:phase-future.embedding-remote-reranker-live]`：(1) **构建** `RemoteRerankerProvider`（镜像 `CrossEncoderReranker` by-index 映射 + `RemoteEmbeddingProvider` ureq/纯函数结构）+ 建 `select_reranker` 工厂（镜像 `select_provider`）+ 纯函数契约测试 + env-gated remote live quality harness，在同一作者手工标注集上比较 real remote reranker vs identity（model-free no-semantic-signal）基线的 MRR / recall@1；(2) 对一个**真实远程 reranker 端点**（SiliconFlow `https://api.siliconflow.cn/v1/rerank` + `Qwen/Qwen3-VL-Reranker-8B`，与 embedding 同 URL 同 key）跑出真实质量数（lead 已 de-risk 证明 round-trip 可行：`config_save` `relevance_score=0.7356` 排 #1、约 46x 区分度、HTTP 200）；(3) Go 新 `RerankerConfig`（`Enabled`/`Provider`/`Endpoint`/`Model`）add-only + 新 `setRerankerEnv` 跨进程 env-bridge（镜像 `setRemoteEnv`）+ Rust 新 `reranker_from_env()` 在 `server.rs`/`data_plane/search.rs` 生产路径首次 opt-in `with_reranker`（`CONTEXTFORGE_RERANKER_PROVIDER` 非空 / 非 `none` 时 → `select_reranker` → `with_reranker`；**默认 unset = 字节等价无 rerank、向后兼容**）。默认构建保持 0 网络 / 0 远程 dep、`reranker-remote` opt-in、API key env-only 永不进 config（ADR-004/008），0 新 dep / 0 proto / 0 migration；既有 `cargo-test` / `go-test` / `lint` / `spec-lint` / `feature-build` 门不退化。

## 2. 业务价值

兑现 ADR-026 + ADR-042 follow-up + roadmap §3.20 一路刻意延后的 `[SPEC-DEFER:phase-future.embedding-remote-reranker-live]`——把「reranker trait + identity/cross-encoder 已实现、但远程 reranker 从未存在、reranker 从未在生产路径接线、真实远程 rerank 质量从未跑出」推进到「构建远程 reranker provider + 真实端点实测质量 + reranker 数据面首次 opt-in 接线」：

- **远程 reranker 从未存在、真实 rerank 质量从未跑出**：Phase 21（ADR-026）确立 reranker 抽象（trait + identity 默认 + 本地 cross-encoder），但远程 reranker provider（cross-encoder over HTTP）从未落地、真实 live 端点 rerank 质量数（MRR / recall@1 over real remote reranker）一路诚实未跑出 `[SPEC-DEFER:phase-future.embedding-remote-reranker-live]`。本 phase **构建** `RemoteRerankerProvider` + `select_reranker` 工厂、对真实端点跑出真实 MRR / recall@1，使「remote reranker 真能把相关文档排到 #1」从假设变为实测事实（lead de-risk 已证明 round-trip 可行：相关文档 `relevance_score=0.7356` 排 #1、约 46x 区分度）。
- **reranker 从未在生产路径生效**：`server.rs`（hybrid :346 + semantic :391）与 `data_plane/search.rs`（semantic :285-287）至今只 `.with_embedder().with_vector_searcher()`、从不 `.with_reranker()`——reranker 仅 Phase 21 的测试 seam（`retriever/mod.rs:630`）生效，生产检索 0 rerank。本 phase 把 reranker 从 config 在生产数据面**首次 opt-in 接线**（`reranker_from_env()` → `with_reranker`），**默认 unset = 字节等价无 rerank（向后兼容）**。这是为何「只加 Go `[reranker]` config 桥而数据面不消费」不诚实——故 task-38.2 端到端：Go 桥 + Rust 数据面消费。
- **in-repo 无 remote rerank 质量证据**：仓库此前只有 model-free `IdentityReranker` + feature-gated 本地 `CrossEncoderReranker` + reranker 测试 seam，无任何 remote live rerank 质量证据。本 phase 以真实测量取而代之（ADR-013，合成-quality REJECTED；de-risk 探针数为可行性证据非质量断言）。
- **最 surgical 的「兑现而非重写」**：reranker trait / identity / by-index 映射范式（`CrossEncoderReranker`）+ ureq/纯函数范式（`RemoteEmbeddingProvider`）+ 工厂范式（`select_provider`）+ env-bridge 范式（`setRemoteEnv`）+ opt-in seam（`with_reranker`）均已存在，本 phase 复用全部既有范式只加 remote provider + 工厂 + harness + Go 桥 + 数据面接线，无新机制。

## 7. 阶段级风险

- **R1（高）remote 是付费外部 API、CI 无密钥导致 live rerank 质量无法在 CI 验证（与 qdrant 的结构性差异）**：qdrant 有免费 OSS service container 可在每次 CI run 验证召回；remote reranker 是付费外部 API、无免费 service container，CI 无密钥时只能 honest-defer skip。
  - **缓解**：task-38.1 harness env-gated 于 `CONTEXTFORGE_RERANKER_API_KEY`——未设时 honest-defer 干净跳过（不 fail），CI 无密钥不 false-red；真实 rerank 质量由**本机已认证 run** 实测并据实回填（lead de-risk 已证明 round-trip 可行：相关文档 `relevance_score=0.7356` 排 #1、约 46x 区分度、HTTP 200）；非网络契约 + well-formed 守护测试无 key 也跑（provider 契约 + 工厂路由 + 标注集结构始终被 CI 验证）；release docs 诚实记录此 CI honest-defer 与 qdrant 的差异（ADR-013，不伪造 CI live 绿）。CI live rerank 质量据实延后、reranker 复用既有 [SPEC-DEFER:phase-future.embedding-remote-ci-credential]（不另造 reranker-ci-credential）。stop-condition：若试图在 CI 伪造 remote live 绿或预填合成质量数则越界、AC1 live 维度不标 `[x]`（守护维度 TEST-38.1.2 达成则部分 ratify，ADR-013）。
- **R2（高）真实 rerank 质量低于 floor / remote 未超 identity / floor 设置失当**：remote reranker 质量可能低于预设 floor（`MRR_remote >= 0.70`），或 remote 未超 identity no-semantic-signal 基线，或 floor 设得过松失去护栏意义。
  - **缓解**：task-38.1 floor 为**文档化护栏**、真实数被 `eprintln!` 报告并据本机已认证 run 回填（ADR-013，绝不预填）；作者标注集故意埋近义干扰使命中非词面 trivial；基线 = identity no-semantic-signal（候选携统一 / 无相关性先验 score、决定性 tie-break ≈ chance），让「remote cross-encoder 超 no-signal 基线」成可证命题（lead de-risk 已证明 round-trip 可行 + 约 46x 区分度）；若真实质量 < floor 则先据真实数调 floor 或诊断（模型 / 标注 / wire format），**不伪造质量数充门**。stop-condition：若真实质量 < floor 且无法据真实数合理校准则如实记录受阻、不强标 AC1 live 维度 `[x]`（守护维度 TEST-38.1.2 达成则部分 ratify，ADR-013）。
- **R3（中）provider/harness 误破默认构建 0-network / 引入新 dep / 泄漏 api_key**：remote live 主题易诱导在默认构建拉入 HTTP/远程依赖、或在 provider / harness / 日志 / config 泄漏 api_key。
  - **缓解**：task-38.1 `RemoteRerankerProvider` 在 `#![cfg(feature = "reranker-remote")]` 模块、harness `#![cfg(feature = "reranker-remote")]` 令默认构建不编译；`reranker-remote = ["dep:ureq"]` 复用 task-22.3 既有 optional `ureq`（0 新 dep，ADR-008）；默认 `cargo test --workspace` 不含该 test、0 网络 / 0 远程 dep（ADR-004）；api_key 仅经 Authorization header、provider `Debug` / harness / config 均不打印不持久化（PRD 安全基线，镜像 `RemoteEmbeddingProvider::Debug`）；task-38.2 `setRerankerEnv` 不处理密钥、`RerankerConfig` 无 api_key 字段（API key env-only 永不进 config.toml）。stop-condition：若默认构建引入网络 / 远程 dep / 新 dep，或 api_key 进 config / 被记录则越界、AC 不标 `[x]`。
- **R4（中）数据面 opt-in 接线破坏向后兼容 / 误溯改 ADR-026 或 ADR-042 D-body**：reranker 数据面首次接线若非严格 opt-in（默认 unset 时仍 rerank）会破坏向后兼容、改变当前检索行为；兑现 ADR-026/042 的 defer 时易直接编辑其 D-body / Ratification。
  - **缓解**：task-38.2 `reranker_from_env()` 严格 opt-in——`CONTEXTFORGE_RERANKER_PROVIDER` 非空 / 非 `none` 才返 `Some`、否则 `None`；`server.rs`/`data_plane/search.rs` 仅 `Some` 时 `with_reranker`，**默认 unset → 字节等价无 rerank（当前行为不变）**，由 TEST-38.2.3 守（unset → 无 rerank provenance marker）；task-38.3 仅以 **add-only Phase-38 Amendment** 标 ADR-026/042 维度 fulfilled，**不溯改其 D-body / 不溯改 `## Ratification`**（ADR-014 D5）；ADR-043 为本 phase 新主 ADR。stop-condition：若默认 unset 时数据面仍 rerank（破坏向后兼容）则 AC2 不标 `[x]`；若溯改 ADR-026/042 任何既有正文 / Ratification 则越界改为 add-only Amendment。
