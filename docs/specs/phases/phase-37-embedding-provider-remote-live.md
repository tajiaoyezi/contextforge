## 1. 阶段目标

v0.29.0 ship 后，ContextForge 的 embedding provider 抽象自 Phase 22 已**全实现且 remote 纯函数契约经测试证明**（`select_provider` "remote" 分支 + `build_request_body`/`parse_response` 纯函数 + ureq `embed`），唯一缺口是 ADR-027 记的 `[SPEC-DEFER:phase-future.embedding-provider-remote]`——**真实远程端点端到端联调 + 实测语义召回数**因「CI 无远程 API 密钥」一路诚实延后，remote 从未对 live 端点测过真实召回。本 phase 关闭该缺口：(1) 建一个 env-gated remote live recall harness，在同一作者手工标注语义集 + 同一 `BruteForceVectorBackend` 精确余弦路径上，比较 real remote 模型 vs deterministic（model-free）基线的 recall@1 / recall@3；(2) 对一个**真实远程端点**（SiliconFlow OpenAI-compatible + `Qwen/Qwen3-Embedding-8B`，dim 1024）跑出真实召回数（lead 已 de-risk 证明 round-trip 可行 + CJK 正常 + MSVC 编译跑通）；(3) 兑现 `factory.rs:52` 注释 "config plumbing is a follow-up"——Go `RemoteProviderConfig` add-only 补 `Model` 字段 + 新 `setRemoteEnv` 跨进程 env-bridge 接通 spawned core daemon。默认构建保持 0 网络 / 0 远程 dep、`embedding-remote` opt-in、API key env-only 永不进 config（ADR-004/008），0 新 dep / 0 migration / 0 provider 核心逻辑改动；既有 `cargo-test` / `go-test` / `lint` / `spec-lint` / `feature-build` 门不退化。

## 2. 业务价值

兑现 ADR-027 + roadmap §3.19 一路刻意延后的 `[SPEC-DEFER:phase-future.embedding-provider-remote]`——把「remote embedding 纯函数契约经测试证明、但 live 端点从未联调、真实召回从未跑出」推进到「真实端点实测召回 + config env-bridge 接通」：

- **remote 真实召回从未跑出**：Phase 22（ADR-027）确立 remote provider 抽象成立（`build_request_body`/`parse_response` 纯函数 fixture 契约测试 + ureq `embed`），但真实 live 端点召回数（recall@k over real remote 模型）因 CI 无密钥一路诚实未跑出 `[SPEC-DEFER:phase-future.embedding-provider-remote]`。本 phase 对真实端点跑出真实 recall@k，使「remote 模型真能高质量语义召回」从契约假设变为实测事实（lead de-risk 已证明 round-trip 可行 + CJK 正常 + MSVC 编译跑通）。
- **in-repo 无 remote 召回证据**：仓库此前只有 deterministic 缺省 provider + feature-gated fastembed + remote 纯函数契约测试（fixture），无任何 remote live 召回质量证据。本 phase 以真实测量取而代之（ADR-013，合成-recall A1 REJECTED）。
- **config 未接通 remote**：`factory.rs:52` 注释 "config plumbing is a follow-up" 标明 remote endpoint/model 仅由 env 读取、config.toml 未接通。本 phase 兑现：Go `[remote]` 段 add-only 补 `Model` + `setRemoteEnv` 跨进程 env-bridge 镜像 Phase 34 的 `setVectorEnv`，让 `[remote]` 配置经 spawned core daemon 既有 env 路径生效（env-wins / 无 [remote] 段行为不变 / API key 永不进 config）。
- **0 provider 核心改动、纯 harness + config-bridge + closeout**：embedding provider 抽象自 Phase 22 已全实现，本 phase 不改 1 行 provider 核心逻辑，只加 env-gated harness（复用既有 `select_provider`/provider）+ Go config env-bridge（add-only `Model` + `setRemoteEnv`）+ closeout，是最 surgical 的「兑现而非重写」。

## 7. 阶段级风险

- **R1（高）remote 是付费外部 API、CI 无密钥导致 live recall 无法在 CI 验证（与 qdrant 的结构性差异）**：qdrant 有免费 OSS service container 可在每次 CI run 验证召回；remote embedding 是付费外部 API、无免费 service container，CI 无密钥时只能 honest-defer skip。
  - **缓解**：task-37.1 harness env-gated 于 `CONTEXTFORGE_REMOTE_API_KEY`——未设时 honest-defer 干净跳过（不 fail），CI 无密钥不 false-red；真实召回由**本机已认证 run** 实测并据实回填（lead de-risk 已证明 round-trip 可行 + CJK 正常 + MSVC 编译跑通）；非网络 well-formed 守护测试无 key 也跑（标注集结构始终被 CI 验证）；release docs 诚实记录此 CI honest-defer 与 qdrant 的差异（ADR-013，不伪造 CI live 绿）。CI live recall 据实延后 [SPEC-DEFER:phase-future.embedding-remote-ci-credential]。stop-condition：若试图在 CI 伪造 remote live 绿或预填合成召回数则越界、AC1 live 维度不标 `[x]`（守护维度 TEST-37.1.2 达成则部分 ratify，ADR-013）。
- **R2（高）真实 recall 低于 floor / remote 未超 deterministic / floor 设置失当**：remote 模型语义召回可能低于预设 floor（`r3 >= 0.70`），或 remote recall@1 未超 deterministic 基线，或 floor 设得过松失去护栏意义。
  - **缓解**：task-37.1 floor 为**文档化护栏**、真实数被 `eprintln!` 报告并据本机已认证 run 回填（ADR-013，绝不预填）；作者标注集故意埋近义干扰使命中非词面 trivial、对照同一 `BruteForceVectorBackend` 精确余弦路径消除 backend 变量；real vs deterministic 同路对照让「remote 超 model-free 基线」成可证命题（lead de-risk 已证明 round-trip 可行）；若真实 recall < floor 则先据真实数调 floor 或诊断（模型 / dim / 标注），**不伪造召回数充门**。stop-condition：若真实 recall < floor 且无法据真实数合理校准则如实记录受阻、不强标 AC1 live 维度 `[x]`（守护维度 TEST-37.1.2 达成则部分 ratify，ADR-013）。
- **R3（中）harness 误破默认构建 0-network / 引入新 dep / 泄漏 api_key**：remote live 主题易诱导在默认构建拉入 HTTP/远程依赖、或在 harness / 日志 / config 泄漏 api_key。
  - **缓解**：task-37.1 `#![cfg(feature = "embedding-remote")]` 令 harness 默认构建不编译；`ureq` 自 task-22.3 已 optional（0 新 dep，ADR-008）；默认 `cargo test --workspace` 不含该 test、0 网络 / 0 远程 dep（ADR-004）；api_key 仅经 Authorization header、harness / `RemoteEmbeddingProvider::Debug` / config 均不打印不持久化（PRD 安全基线）；task-37.2 `setRemoteEnv` 不处理密钥、`RemoteProviderConfig` 无 api_key 字段（API key env-only 永不进 config.toml）。stop-condition：若默认构建引入网络 / 远程 dep / 新 dep，或 api_key 进 config / 被记录则越界、AC 不标 `[x]`。
- **R4（中）误溯改 ADR-027 D-body / `## Ratification`，或 config-bridge 破坏向后兼容**：兑现 ADR-027 的 `[SPEC-DEFER:phase-future.embedding-provider-remote]` 时易直接编辑其 D-body / Ratification；`setRemoteEnv` 若非 env-wins / 在无 [remote] 段时导出会破坏向后兼容。
