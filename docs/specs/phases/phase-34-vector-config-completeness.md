## 2. 业务价值

vector-backend 配置故事收口补全——补齐「vector-dim 自动协商、vector-backend 配置文件」两项真实补全，且经 grounding 对 survey 夸大的「`get_source_chunk` 隔离 gap」据实诚实校正为 verify-only（这是本 phase 的 ADR-013 核心价值）。Phase 31/33 两轮治理债清扫后绿色 backlog 已薄，本 phase 据实保持**小**而非凑数（ADR-013）：

### 34.1 vector-dim auto-negotiation（vector-dim-auto-negotiation，🟢）

- `core/src/retriever/vector/factory.rs:33-39` `select_vector_backend(name, dim)` 体内 `let _ = dim;`（:39）**静默丢弃**已解析的 `CONTEXTFORGE_VECTOR_DIM`：`server.rs` `resolve_vector_backend`（:540）/ `parse_vector_backend`（:551-559）从 env 解析 `(name, dim)` 并把 `dim` 传入 factory，但 factory 当前不消费它——与 `core/src/embedding/factory.rs:81-96` `negotiate_dim(provider_dim, requested)`（`requested != 0 && provider_dim != requested → DimMismatch{expected: requested, got: provider_dim}`，:88-96）的诚实协商形态不对称。
- 本 phase 补对称协商 seam：`VectorBackend` trait（`traits.rs:11-16`）加 `expected_dim(self) -> Option<usize>`，带 **默认 impl 返回 None**（dim-agnostic），`BruteForceVectorBackend` 沿用 None；factory 以 `negotiate_vector_dim(dim, backend.expected_dim())` 替换 `let _ = dim`。`negotiate_vector_dim` 为纯函数：`requested==0` → Ok；`declared==None` → Ok；`requested == Some(declared)` → Ok；非零 `requested != Some(declared)` → `VectorError::DimMismatch{expected: requested, got: declared}`（复用既存 `types.rs:83` 的 variant，**0 新 type / 0 新 dep**）。`VectorStore` blanket impl（`traits.rs:57-59`）不动（默认 method add-only，ADR-014 D5）。
- **HONEST CAVEAT（不夸大）**：default BruteForce `expected_dim()=None` → 在 default build 协商对任意 dim 返回 Ok（**无 enforcement、byte-equivalent 默认行为**，ADR-004）；真实 enforcement 只对声明 dim 的 feature backend（qdrant/lancedb/sqlite-vec）生效，其 live exercise 须 feature build → honest-defer `[SPEC-DEFER:phase-future.vector-dim-feature-enforce]`（如实陈述，不当作 default build 已生效宣称）。交付物 = `CONTEXTFORGE_VECTOR_DIM` 不再被静默丢弃 + 纯函数协商 seam 单测。镜像 `embedding::factory::negotiate_dim` 的协商测试形态。

### 34.2 vector-backend config-file（vector-backend-config-file，🟢）

- vector backend 今为 **env-only**：`CONTEXTFORGE_VECTOR_BACKEND`/`CONTEXTFORGE_VECTOR_DIM` 仅由 Rust `server.rs` `resolve_vector_backend`（:540）读，`config.toml` 无对应段。Rust core 有 `serde`/`serde_json` 但**无 `toml` dep**——在 Rust 端加配置读取器会破 0-dep（ADR-008）。
- Go `internal/config/config.go` 已用手写 TOML codec 解析 `config.toml` 的 `[collections]`/`[remote]`/`[embedding]` 段（`EmbeddingConfig` :61-64 + `encodeTOML` :194-196 + `decodeTOML` section switch :219-221 + `assignEmbedding` :322-338），且 spawn 的 Rust daemon **继承 Go 进程 env**（`internal/daemon/daemon.go:201-209` `launch` 的 `exec.Command(...)` 未设 `cmd.Env` → 继承 `os.Environ()`；`cmd/contextforge/main.go:254-268` `setDataDirEnv` 即以 `LookupEnv` 旧值 + 仅非空时 `Setenv` + 还原闭包同形导出 `CONTEXTFORGE_DATA_DIR`）。
- 本 phase 补：`config.Config` 加 `[vector]` 段（`Vector VectorConfig`，字段 `Backend string` toml `backend` tag / `Dim int` toml `dim` tag，镜像 `EmbeddingConfig`）+ encode/decode/assign 三处范本 + `setVectorEnv` helper（镜像 `setDataDirEnv`）在 `[vector]` 存在**且对应 env 未已设**时导出 `CONTEXTFORGE_VECTOR_BACKEND`/`CONTEXTFORGE_VECTOR_DIM`，由 spawn 的 core daemon 经既有 `resolve_vector_backend` env 路径拾取。
- **ENV WINS（后向兼容）**：显式设置的 env 覆盖配置文件（`setVectorEnv` 仅在 `LookupEnv` 报未设时导出）；**无 `[vector]` 段 → 不导出 → unset → BruteForce byte-equivalent**（ADR-004 默认不变）。这是与 `CONTEXTFORGE_DATA_DIR` 同一已验证的跨进程 env-bridge（**非** `daemon.Options.DataDir` field 重构——那须改 spawn 契约 + thread 子进程 env，留 `[SPEC-DEFER:phase-future.daemon-options-datadir]`，承 Phase 33 同形 defer）。Rust 0-dep 保持（toml 仅 Go 端，ADR-008）。

## 7. 阶段级风险

- **R1（中）vector-dim 协商误破默认行为 / `expected_dim` 默认 impl 不覆盖既有 backend**：factory 以 `negotiate_vector_dim` 替换 `let _ = dim` 若协商逻辑或 `expected_dim` 默认值有偏，会令 default build 误报 `DimMismatch` 或破 BruteForce byte-equivalent。
  - **缓解**：task-34.1 `expected_dim` 默认 impl 返回 None（dim-agnostic）→ `negotiate_vector_dim` 对 `declared==None` 一律 Ok（含 BruteForce 任意 dim）；纯函数严格镜像 `embedding::factory::negotiate_dim`（仅非零 `requested != Some(declared)` 报错）；TEST-34.1.1 断言 0/None/matching=Ok、mismatch=DimMismatch；TEST-34.1.2 断言 BruteForce default 任意 dim 被接受 + byte-equivalent。stop-condition：default build 误报 DimMismatch / 既有 29.1.* / 32.* 测试退化则 AC1 不标 `[x]`。
- **R2（中）Go `[vector]`→env 桥破既有 config round-trip / env-wins 失守**：`config.Config` 加 `[vector]` 段须 add-only 不破既有 `[collections]`/`[remote]`/`[embedding]` round-trip；`setVectorEnv` 若在 env 已设时仍覆盖会破后向兼容。
  - **缓解**：task-34.2 `[vector]` 段镜像 `EmbeddingConfig` add-only（encode/decode/assign 三处对称，既有段 encode/decode 不动，ADR-004 既有契约不变）；`setVectorEnv` 严格镜像 `setDataDirEnv` 的 `LookupEnv` 旧值守护——仅在 `!hadOld`（env 未设）时 `Setenv`（env-wins）；TEST-34.2.1 断言有/无段 round-trip + 既有段不受影响；TEST-34.2.2 断言段存在导出 / env 已设不覆盖 / 无段不导出。stop-condition：既有 config round-trip 退化 / env-wins 失守 / 无段非 byte-equivalent 则 AC2 不标 `[x]`。
- **R3（中）`get_source_chunk` 隔离误重实现既存对称性**：survey 暗示 `get_source_chunk` 须补 workspace 隔离，易诱导改 `search.rs:417-431` 既已正确的 scope 逻辑，但该隔离自 task-12.2 已存在。
  - **缓解**：task-34.3 据 grounding（`search.rs:421-423` 非空 `req.workspace_id` → 仅该 workspace、空 → aggregate-all）只交付 **verify-only 守护测试**（workspace_id 设 / 跨 workspace / 空 三态），**不写任何新代码**（CLAUDE.md 简单优先；ADR-013 不为已实现的对称性重写实现）；在 ADR-039 D3 记 grounding 校正。stop-condition：若改 `get_source_chunk` 既有 scope 逻辑则越界——task-34.3 交付物收敛为 verify-only 守护测试。
- **R4（低）Rust 端误引 toml 破 0-dep**：vector 配置故事易诱导在 Rust core 加 `config.toml` 直读器（须引 `toml` dep）。
  - **缓解**：task-34.2 配置读取留在 Go 端既有手写 TOML codec，跨进程经 `CONTEXTFORGE_DATA_DIR` 同形 env-bridge 桥到 Rust（`resolve_vector_backend` env 路径已在）；Rust 0-dep 保持（ADR-008）；Rust 端直读 honest-defer `[SPEC-DEFER:phase-future.rust-side-toml-config-reader]`。stop-condition：若 Rust 端引 toml dep 则越界。
