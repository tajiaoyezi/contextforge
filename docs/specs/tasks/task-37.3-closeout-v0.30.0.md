## 1. Background

Phase 37（embedding-provider-remote-live）兑现一个自 Phase 22 起被 honest-defer 的真实维度：`RemoteEmbeddingProvider`（`core/src/embedding/remote_provider.rs`）的 `build_request_body` / `parse_response` 纯函数构造层 + `select_provider` 工厂 `"remote"` arm（`core/src/embedding/factory.rs`）自 Phase 22 起即**已实现**，但**真实远程 embedding 端点的端到端联调 + 实测语义召回**因仓库无可用远程端点而被诚实延后（ADR-027 `[SPEC-DEFER:phase-future.embedding-provider-remote]`）；在此之前仓库只有 deterministic 缺省 provider + feature-gated fastembed，remote 仅有契约层 + 纯函数契约测试（`build_request_body` / `parse_response`），从未对 live 端点测过真实召回。Phase 37 关闭它：(1) 建一个 env-gated live recall harness，在同一标注集 + 同一 `BruteForceVectorBackend` 精确余弦路径上比较 real remote 模型 vs deterministic 基线的 `recall@1` / `recall@3`；(2) 对一个**真实**远程 embedding 端点跑（DE-RISK 已被主 agent 真实证实：SiliconFlow `https://api.siliconflow.cn/v1/embeddings`（OpenAI-compatible）+ 模型 `Qwen/Qwen3-Embedding-8B` 端到端 round-trip 跑通，native dim=4096、OpenAI 风格 `dimensions` 参数被接受并生效（MRL，本 phase 用 1024）、CJK 输入正常；Rust `RemoteEmbeddingProvider` → ureq → `parse_response` 路径在 Windows MSVC 下 `--features embedding-remote` 真实编译并跑通，返回 1024 维向量）；(3) 把 remote provider config 从 env-only 推进到 `[remote]` config.toml 桥接（`setRemoteEnv` 镜像 Phase 34 `setVectorEnv`）。默认构建不变：embedding-remote opt-in，默认构建 0-network / 0 新 dep（ADR-004/008）；0 新 dep（ureq 自 task-22.3 起即 optional）。
