## 1. Background

Phase 38（embedding-remote-reranker-live）兑现一个自 Phase 14 的 reranker abstraction（ADR-026）起被 honest-defer 的真实维度：母 ADR-026 在 v0.14.0 确立 `Reranker` trait（`core/src/rerank/traits.rs`）+ `IdentityReranker`（`core/src/rerank/identity.rs`）+ `CrossEncoderReranker`（`core/src/rerank/cross_encoder.rs`，feature `reranker-fastembed`），但**远程 reranker 端到端联调 + 实测 rerank 质量**从未交付——与 embedding 维度不同（`RemoteEmbeddingProvider` 自 Phase 22 已全实现，Phase 37 仅复用既有 provider 跑 live recall），**`RemoteRerankerProvider` 与 `select_reranker` 工厂此前并不存在**，故 Phase 38 须先「构建」provider + 工厂 + 契约测试 + live harness。此外 reranker 自 Phase 21 起仅是 opt-in builder seam（`.with_reranker(..)` 仅测试用）——生产数据面（`server.rs` hybrid `:334` + semantic `:376` 两处、`data_plane/search.rs` semantic `:282` 一处）至今只调 `.with_embedder().with_vector_searcher()`、**从不调 `.with_reranker(..)`**，故 reranker 从未由 config 在生产路径接线。Phase 38 关闭它：(1) 新建 `RemoteRerankerProvider` + `select_reranker` 工厂 + 契约测试；(2) 建一个 env-gated live rerank harness，在作者手工标注 query×candidate 集上比较 real remote cross-encoder vs `IdentityReranker`（no-semantic-signal）基线的 `MRR` / `recall@1`；(3) 对一个**真实**远程 reranker 端点跑（DE-RISK 已被主 agent 真实证实：SiliconFlow `https://api.siliconflow.cn/v1/rerank`（与 embedding 同 URL 同 key、仅 model 不同）+ 模型 `Qwen/Qwen3-VL-Reranker-8B` 端到端探针通过：查询「如何保存配置到文件」对 4 文档 rerank，相关文档 `config_save` `relevance_score=0.7356` 排 #1、近义干扰 `config_load=0.0158` 排 #2、无关项 `~0.0006`/`~0.0003`，HTTP 200——端点可用 + 排序语义正确 + 约 46x 区分度；映射回 `candidates[index]`、set `score=relevance_score`、annotate reason，镜像 `CrossEncoderReranker` 的 by-index 映射）；(4) 把 reranker 从 config 在生产数据面首次 opt-in 接线（新 `reranker_from_env()` → `select_reranker` → `.with_reranker(..)`）+ 把 reranker config 从无桥接推进到 `[reranker]` config.toml 桥接（`setRerankerEnv` 镜像 `setRemoteEnv`/`setVectorEnv`）。默认构建不变：reranker-remote opt-in，默认构建 0-network / 0 新 dep（ADR-004/008）；0 新 dep（ureq 自 task-22.3 起即 optional）；数据面默认 unset → 不接 reranker → 字节等价无 rerank（向后兼容，当前行为不变）。

两个实现 task 全 Draft（实施授权另行）：38.1（remote-reranker-provider-and-live-recall——新建 `RemoteRerankerProvider` + `select_reranker` 工厂 + 契约测试 + 新增 `core/tests/remote_rerank_recall.rs`，`#![cfg(feature = "reranker-remote")]`，env-gated 在 `CONTEXTFORGE_RERANKER_API_KEY`（未设 → 干净 honest-defer skip 不 fail）；作者手工标注 query×candidate 集（埋近义干扰）；real remote cross-encoder vs `IdentityReranker`（no-semantic-signal）基线 `MRR` / `recall@1`，floor `MRR_remote >= 0.70` 且 `MRR_remote > MRR_identity`；另有非网络契约 + well-formed 守护测试无 key 也总是跑；新增 feature `reranker-remote = ["dep:ureq"]`，0 新 dep）/ 38.2（reranker-config-bridge-and-data-plane-wiring——`internal/config/config.go` 新 `RerankerConfig`（无 api-key 字段）+ 新 `setRerankerEnv` helper（`[reranker]` 段 → 导出三 env，env-wins，API key env-only 永不进 config，接线进 doServe + doMCP）+ **Rust 数据面首次 opt-in 接线**（`reranker_from_env()` → `select_reranker` → `.with_reranker(..)` 接 server.rs hybrid+semantic + data_plane/search.rs semantic，默认 unset 字节等价无 rerank））。
