## 1. Background

`core/src/rerank/` 自 Phase 21 起其 reranker 抽象**已完整落地**：`Reranker` trait（`traits.rs:22-34`，object-safe）/ `RerankError`（`traits.rs:40-52`，`#[non_exhaustive]`）/ `IdentityReranker`（`identity.rs`，model-free 决定性默认）/ `CrossEncoderReranker`（`cross_encoder.rs`，feature `reranker-fastembed` 的本地 fastembed BGE-reranker-base）。但 **remote rerank 端点（OpenAI/Cohere/SiliconFlow 风格 HTTP rerank API）的 provider 至今不存在**：`core/src/rerank/` 下**没有** `remote_reranker.rs`、**没有** `select_reranker` 工厂——与 embedding 侧不同（`RemoteEmbeddingProvider` 自 Phase 22 已全实现，`select_provider` 工厂自 task-22.1 已在），reranker 侧的 remote 维度从未构建。ADR-026（reranker-provider）确立 trait + identity + cross-encoder 三件，其 **remote reranker** 维度记为 `[SPEC-DEFER:phase-future.embedding-remote-reranker-live]`（由 ADR-042 / task-37.x spec §范围外 / roadmap follow-up 记录）。Phase 38 关闭这条 gap，本 task 是第一步——**构建** `RemoteRerankerProvider` + `select_reranker` 工厂 + 契约测试，并以一个「real remote cross-encoder rerank 质量 vs `IdentityReranker`（no-semantic-signal）基线」方法学 harness 在 live remote rerank 端点上量真实 rerank 质量：

- **B0 本 task 构建 provider（非「0 provider 改动」）**：与 task-37.1（消费既有 `RemoteEmbeddingProvider` / `select_provider`、0 provider 改动）不同——`RemoteRerankerProvider` 与 `select_reranker` 工厂此前不存在，故本 task = **实现 provider（`core/src/rerank/remote_reranker.rs`）+ 建工厂（`core/src/rerank/factory.rs`）+ 契约测试 + live harness**。provider 镜像 `CrossEncoderReranker`（`cross_encoder.rs:50-97` by-index map back + score 覆写 + provenance 注释）的结构、只把本地 fastembed 模型换成 ureq POST live rerank 端点；工厂镜像 `select_provider`（`factory.rs:27-83`）的 env 读取 + feature-gated 显式 Err 范式。
- **B1 方法学 = real remote cross-encoder rerank 质量 vs `IdentityReranker`（no-semantic-signal）基线（同一标注集）**：在**同一**作者手工标注 query×candidate 集上，用 `select_reranker("remote")` 经 live 端点对候选 rerank、`select_reranker("identity")` 作 no-semantic 基线 rerank，二者各对每个标注 query 重排候选后看其单一已知相关文档落到第几位，`MRR = mean(1/rank_of_relevant)`、`recall@1 = mean(rank_of_relevant == 1)`。基线候选喂入时**携统一 / 无相关性先验 score**（候选不带语义信号——例如全部同分或按插入序），故 `IdentityReranker` 的「score desc + chunk_id asc」决定性 tie-break ≈ chance（无 query×doc 联合信号可用）；real cross-encoder 须用 query×doc **联合**打分把相关文档排到 #1。delta 即「真实 cross-encoder 相对无语义信号基线买到的 rerank 质量」。区别于 task-37.1 量的 embedding **dual-encoder 语义召回**（cosine 召回）——本 task 量的是 **cross-encoder 联合重排质量**（rerank top-1 / MRR），二者维度互补（ADR-026 D1：cross-encoder 比 dual-encoder cosine 更精，rerank 初始 top-k 可抬 top-1 / MRR）。
- **B2 作者手工标注 query×candidate 集（诚实范围，非大基准）**：作者手工标注若干 query，每个 query 配一组候选——其中一个是**已知相关**文档，其余是**故意近义干扰**（如 `config_save` vs `config_load`、`bm25` vs `hybrid`），使「把相关文档排 #1」**非词面 trivial**——必须靠 query×doc 语义联合打分而非共享词。这是**小型手工标注集**（诚实范围），证明真实 cross-encoder 把明显相关文档排在近义干扰之上，**不是**大型标准 rerank 基准（NDCG/MRR 大基准续 `[SPEC-DEFER:phase-future.reranker-large-corpus-quality]`，ADR-013）。
- **B3 honest-defer 守门（CONTEXTFORGE_RERANKER_API_KEY 未设 → skip 干净退出，不 fail）**：live rerank 测试**第一步**读 `CONTEXTFORGE_RERANKER_API_KEY`——未设（CI 无密钥）则 `eprintln!` 一条 skip notice（说明需 live remote rerank 端点 + 设 `CONTEXTFORGE_RERANKER_ENDPOINT/_MODEL/_API_KEY`）+ `return`（测试**干净通过**，**不** fail）；factory 另自 env 读 `CONTEXTFORGE_RERANKER_ENDPOINT/_MODEL/_PROVIDER`，api_key **永不**记录。这样无密钥的本地 / CI 环境**干净 skip 而非红**（honest-defer，ADR-013）；设密钥的 dev-box 才进真实 rerank 质量度量。
- **B4 floor 是 guard、真实值是报告（ADR-013 禁伪造）**：harness 先 `eprintln!` **真实测得**的 identity vs remote MRR/recall@1 + delta，再 `assert!`：floor `MRR_remote >= 0.70`（不退化下界）**且** `MRR_remote > MRR_identity`（real cross-encoder 必须胜过无语义信号基线，否则 remote rerank 买不到东西）。floor 是守门；真实数字在设密钥的真实 run 跑出后回填 §10 + v0.31.0 evidence（Draft 阶段 **待回填**，绝不预填、绝不伪造，ADR-013）。
- **B5 de-risk 已由主 agent 真实验证（真实非合成，可行性证据）**：SiliconFlow rerank 端点（`https://api.siliconflow.cn/v1/rerank`，与 embedding 同 URL 同 key、仅 model 不同）+ 模型 `Qwen/Qwen3-VL-Reranker-8B` 端到端 round-trip 已跑通——主 agent 探针「查询『如何保存配置到文件』对 4 文档 rerank」实测：相关文档 `config_save` `relevance_score=0.7356` 排 #1、近义干扰 `config_load=0.0158` 排 #2、无关项 `~0.0006 / 0.0003`，HTTP 200——证明端点可用 + 排序语义正确 + 约 46x 区分度（`0.7356 / 0.0158`）。请求 `{model, query, documents:[字符串], top_n, return_documents:false}`、响应 `{results:[{index, relevance_score}], meta}`。本 task 把这条 de-risked 路径制度化为可复现的 rerank-quality harness（**de-risk 探针数仅作可行性证据**，真实 MRR/recall 数 task 实测后回填、不预填，ADR-013）。

经核 `ureq` 自 task-22.3 已 optional（`embedding-remote` feature 引入）——本 task `reranker-remote = ["dep:ureq"]` 复用同一可选 dep、**0 新 dep**；`IdentityReranker` 默认可用——本 task **0 新 dep** + **0 schema migration**（纯新增 provider / 工厂 / test 文件，无表）+ **0 默认构建变更**（`RemoteRerankerProvider` / harness 经 `reranker-remote` feature gate，默认 build 0-network-dep / 0-network 不变，ADR-004/008）。
