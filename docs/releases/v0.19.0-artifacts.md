## 6. 版本声明（contract bump 评估）

- trace FTS5 + VACUUM 是 `SqliteTracePersist` add-only 方法（既有 `put`/`get`/`list`/`load_warm` 签名语义不变）。events SSE 是 add-only 新 endpoint（既有 long-poll + 22-endpoint 不变）。proto `SubscribeEventsRequest.since_ts`/`last_event_id` add-only（既有 field 1/2 不动）。event-bus 配置保守默认（1000/不分区/100ms）使既有行为默认不变。
- 默认构建 0 新依赖 / 0 network（ADR-004）。
- 故 **无 breaking contract bump**；既有客户端无需改动。SSE/重放/分区均 opt-in。
