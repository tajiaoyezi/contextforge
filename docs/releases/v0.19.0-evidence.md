# ContextForge v0.19.0 Evidence — observability-hardening (TraceStore FTS5 + VACUUM + events SSE/重放 + event-bus 配置 + ADR-031 ratified)

> v0.19.0 ships Phase 26: the two observability signal paths landed in Phase 16 are hardened. **TraceStore**
> (`core/src/data_plane/search_persist.rs`) gains FTS5 content search (`search_fts`) + periodic VACUUM/prune
> so `search_traces.db` is queryable-by-content and no longer grows unbounded (task-26.1); **events**
> (`internal/consoleapi`) gain an SSE real-time push endpoint (`GET /v1/observability/events/stream`, add-only
> beside the long-poll) + replay of missed memory state-op events from the persistent audit log (task-26.2);
> and the **EventBus** gains capacity / partition / drain-timeout config (task-26.3). The **default build stays
> 0-new-dependency, 0-network** (ADR-004): FTS5/VACUUM reuse rusqlite bundled SQLite, SSE uses Go stdlib
> `http.Flusher`, replay reads the existing `audit_log`, event-bus config reuses the `with_capacity` seam.
> **ADR-031** is ratified `Proposed → Accepted` on the **real non-synthetic** verification of D1–D6; the real
> daemon-served SSE end-to-end (live curl) dimension is **honestly recorded-as-held** per ADR-013 (CI has no
> running daemon). Existing long-poll endpoint + 22-endpoint contract + `put`/`get`/`list`/`load_warm` signatures
> do not regress (add-only, ADR-015 D1).
