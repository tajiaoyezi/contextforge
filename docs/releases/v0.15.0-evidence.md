# ContextForge v0.15.0 Evidence — embedding-provider-completion (provider 配置选择 + 缓存 + 远程骨架 + ADR-027 ratified)

> v0.15.0 ships Phase 22: it grows the Phase 19 embedding layer ("hardcoded `DeterministicEmbeddingProvider`
> default + a single feature-gated `FastEmbedProvider`") into a **configurable provider layer** — runtime
> `select_provider` factory (deterministic / fastembed / remote) with **dim negotiation** (no silent
> truncate/pad), a **content-hash embedding cache** (`CachingEmbeddingProvider`, memory L1 + optional SQLite
> L2), a **feature-gated remote provider skeleton** (`RemoteEmbeddingProvider`, OpenAI/Cohere HTTP), and an
> **opt-in remote-reachability health probe**. The **default build stays local, model-free, and 0-network-dep**
> (deterministic identity provider is the default; fastembed/remote are feature-gated + explicit opt-in —
> ADR-004 local-first). **ADR-027** (embedding-provider-abstraction) is ratified `Proposed → Accepted` on the
> **real non-synthetic** unit/contract verification of D1–D5; remote real-network 联调 / keys / recall quality
> and the real remote health probe are honestly **deferred** (ADR-013 — CI has no credentials/network).

## 5. Upgrade / Rollback

- Upgrade：默认构建行为不变（deterministic 缺省 + 0 模型 / 0 网络 dep），无迁移。`[embedding]` 为 add-only 配置段（缺省 `Provider=""/Dim=0` → deterministic，向后兼容；既有 `[remote]`/`[[collections]]` 不受影响）。embedding 缓存为库装饰器（opt-in 包裹）；远程 provider 需 `--features embedding-remote` + 显式 opt-in 配置 + 环境密钥（默认镜像不含）。
- Rollback：`git tag -d v0.15.0` + 删 GitHub Release / ghcr tag；默认镜像与 v0.14.0 行为兼容（BM25 + 0-dep deterministic 语义/hybrid 路径 + config-only health 探针），回退非破坏。
