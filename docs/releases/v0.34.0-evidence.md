## 5. Upgrade / Rollback

- Upgrade：**首次刻意默认行为变更（非 byte-equivalent，由 ADR-046 承接）**——v0.34.0 起，daemon 经生产索引路径**新建**的 collection 默认绑 `code_cjk` 分词器（代码符号子词 + CJK bigram，更好的代码/CJK 检索召回），不再是 legacy Tantivy `TEXT`。**既有 collection 不受影响**：经 `open_in_dir` 读回持久化 `meta.json` schema，继续以原 analyzer（多为 `TEXT`）索引/检索、不被静默失效。**opt-out 回 legacy `TEXT`**：设环境变量 `CONTEXTFORGE_TOKENIZER=default`，或 config.toml `[retrieval]` 段 `tokenizer = "default"`（env-wins）。**升级既有 collection 到 `code_cjk`**：用户主动经 `IndexSession::reindex_with_tokenizer`（Phase 30 起已备，读 SQLite chunk 真相源重建 Tantivy 索引）；v0.34.0 **不自动迁移**用户数据。jieba 真分词 `cjk_segmenter` 仍需 `--features cjk-segmenter` 构建 + `CONTEXTFORGE_TOKENIZER=cjk_segmenter` opt-in（默认构建不含，0-dep baseline）。`[retrieval]` config 段 + `RetrievalConfig` 为 add-only（既有 config.toml 无 `[retrieval]` → zero value → core 默认 code_cjk）。
- Rollback：`git tag -d v0.34.0` + 删 GitHub Release / ghcr tag。回退到 v0.33.0 后，新建 collection 重回 legacy `TEXT` 默认；v0.34.0 期间新建的 `code_cjk` collection 在 v0.33.0 下仍可检索（v0.33.0 的 retriever 无条件注册 `code_cjk` analyzer，index/query schema-driven 对称——但 v0.33.0 生产索引默认建 `TEXT`，故行为差异仅在新建时的默认绑定）。`embedding_cache` / chunks schema 未改（0 migration）→ v0.33.0 ↔ v0.34.0 数据文件互通。若不希望 v0.34.0 改默认，无需回退——`CONTEXTFORGE_TOKENIZER=default` opt-out 即得 v0.33.0 行为。
