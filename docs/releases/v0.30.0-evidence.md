### 3. 跨 run @1 波动 + CI honest-defer 诚实 caveat（ADR-013，MUST 记录）

**(a) recall@3=1.0000 是「明显语义对排在近义干扰之上」的证明，不是大基准质量断言**：作者手工标注集仅 **15 case / 16 doc**（小型），recall@3=1.0（3 次 run 全稳定）证明 real 远程模型把明显语义对稳定排进 top-3、压过故意埋的硬近义干扰——这是「远程模型语义召回在该标注集上正确」的端到端联调正确性证明，**不**代表大基准 / 大规模标注集质量。大语料语义质量据真实诚实延后 `[SPEC-DEFER:phase-future.embedding-large-corpus-recall]`。

**(b) recall@1 跨 run 波动（0.8667–0.9333，13–14/15）如实记录**：remote 模型 / 服务**非完全确定**（远程服务端可能有非确定性），故 recall@1 在 3 次 run 间波动；那一两个 @1 miss **恰是故意埋的硬近义干扰**（config_save↔config_load、hybrid↔bm25）的 top-1 让位——明显语义对仍稳进 top-3（recall@3=1.0 全稳定）。deterministic 基线 recall@1=0.0000 / recall@3=0.0667（稳定，model-free 无语义信号）。delta@3=+0.9333。如实记录波动，**不**取最高值伪装为单一确定结果。

**(c) CI honest-defer（关键诚实差异，与 qdrant 不同）**：remote embedding 是**付费外部 API、无免费 service container**——与 v0.29.0 的 qdrant **不同**（qdrant 有免费 OSS service container 在**每次 CI run** 守护活体召回）。本 harness 在 CI 无密钥时**干净 honest-defer SKIP**，真实召回**由本机已认证 run 实测**而**非每次 CI run 守护**。据 ADR-013 据实记录，**不**夸大为「CI 每次 run 守护召回」——真实召回数据来源是本机认证 run（SiliconFlow + Qwen3-Embedding-8B，3 次 run），CI 仅护栏「无密钥时干净 SKIP 绝不 fail」+ harness 编译/结构门。
