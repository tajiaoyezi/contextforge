# ContextForge v0.16.0 Evidence — vector-persistence-and-cross-platform (hnsw 持久化 + sqlite-vec MSVC + ADR-028 ratified)

> v0.16.0 ships Phase 23: it makes the feature-gated vector backends **persistent + cross-platform** —
> **hnsw graph persistence** (`HnswBackend::save`/`load` to `VectorIndexConfig.persistence_path` +
> rebuild-on-load fallback, `vector-hnsw`), a **sqlite-vec Windows MSVC cross-platform** investigation
> that **resolved the Phase 18 MSVC-build-blocked stop-condition** (real build + run on
> `x86_64-pc-windows-msvc`), and a **vector incremental-index evaluation** (brute-force / sqlite-vec
> support row-level append; hnsw full-rebuild deferred). The **default build stays local, 0-vector-dep,
> and BM25-baseline-unchanged** (all persistence/cross-platform capability is feature-gated — ADR-023
> D5). **ADR-028** (vector-persistence-strategy) is ratified `Proposed → Accepted` on the **real
> non-synthetic** verification of D1–D4; honest caveats and deferrals are recorded per ADR-013.
