# ContextForge v0.1.0 Artifact Manifest

**版本**：`v0.1.0`
**日期**：2026-05-23
**依据**：ADR-007 `minimal-tarball-distribution`

## GitHub Release Tarball

目标文件名：

```text
contextforge-linux-amd64.tar.gz
```

必含条目：

- `contextforge`（Go control-plane binary，executable）
- `contextforge-core`（Rust data-plane binary，executable）
- `contextforge.example.toml`
- `README.md`
- `LICENSE`

仓库内 release contract：

- `internal/release.BuildTarball`
- `internal/release.ValidateTarball`
- `internal/release.ValidateSmokeEvidence`
- `internal/release.CheckBenchmark`
- `scripts/release_smoke.sh`

## Source Self-host

源码自托管入口：

```bash
git clone https://github.com/tajiaoyezi/contextforge.git
cd contextforge
git checkout v0.1.0
go mod download
cargo fetch
bash scripts/release_smoke.sh
```

## Docker Compose

ADR-007 记录 v0.1 分发形态包含 Docker Compose；当前 tag 的可验证交付物是 tarball contract + source self-host + release smoke gate。Docker Compose 发布作业尚未落为可验证文件，后续应在 release CI 中补齐并复用 `scripts/release_smoke.sh`。

## Release Gate

发包前必须通过：

```bash
bash -lc 'source docs/s2v/scripts/lib/preflight.sh; source docs/s2v/scripts/lib/verify.sh; s2v_baseline_green "cmd/contextforge internal core/src core/tests"'
bash scripts/release_smoke.sh
```

已验证结果见 `docs/releases/v0.1-evidence.md`。
