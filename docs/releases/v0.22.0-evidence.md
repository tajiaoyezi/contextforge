### 3. lancedb 真实 ANN 索引 + compaction + 多 backend 选择矩阵（task-29.3，🟡 feature build）

`cargo test -p contextforge-core --features vector-lancedb --lib retriever::vector::lance_db` → **4 passed; 0 failed**（CI 默认不构建该 feature，`--lib` scoped 规避 rustc 1.95.0 broad-test ICE）：

- TEST-29.3.1 真实多 backend 选择矩阵（deterministic clustered 语料 n=1024 dim=384 32 query，ground-truth=exact flat；单次代表值）：

  | backend | recall@5 | recall@10 | build | query/q | caveat |
  |---|---|---|---|---|---|
  | brute-force（0-dep，exact） | 1.0000 | 1.0000 | 0 | ~5.5 ms | 默认，精确，无 dep；O(n) 小/中语料最优 |
  | lancedb flat（exact，ground-truth） | 1.0000 | 1.0000 | — | ~7.6 ms | 嵌入式列存，无索引 |
  | lancedb IVF_PQ（parts=16, subs=16） | ~0.33–0.39 | ~0.41–0.46 | ~2.8 s | ~51 ms | 有损 PQ；modest n 下慢且召回折损大；run-to-run 抖动（PQ 训练未播种） |
  | lancedb IVF_HNSW_SQ（m=16, ef=100） | ~0.89 | ~0.90 | ~0.25 s | ~3.5 ms | 高召回 + 最快索引；lancedb ANN 首选档 |

- **真实结论（ADR-013 据实）**：① modest n（~1024）下 brute-force exact 既最准又最快 → **实测背书 ADR-023 D5 默认 0-dep BruteForce**（重型 ANN 开销大语料才回本）；② lancedb ANN 内 IVF_HNSW_SQ（高召回·快）> IVF_PQ（重压缩·modest n 下劣）；③ IVF_PQ 小语料召回折损 + 查询变慢是真实有损权衡（非 bug）。大语料拐点 `[SPEC-DEFER:phase-future.vector-large-corpus-perf]`。
