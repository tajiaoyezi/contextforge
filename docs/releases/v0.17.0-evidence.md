# ContextForge v0.17.0 Evidence — code-and-cjk-tokenizer-and-eval-hardening (opt-in tokenizer + eval 校验器 + ADR-029 ratified)

> v0.17.0 ships Phase 24: an **opt-in code/CJK tokenizer** (`content` field splits
> camelCase/snake_case/dotted.path/kebab-case while preserving the original token + CJK bigram, task-24.1)
> and a **hardened eval ruler** (golden-dataset validator `ValidateGoldenSemantic` + a code/CJK-annotated
> `golden-semantic.jsonl`, task-24.2), with a **real before/after recall delta** measured through the
> production `Retriever` BM25 path (task-24.3). The **default build stays 0-new-dep, default-tokenization-
> unchanged, eval-gate-threshold-unchanged** (the tokenizer is opt-in via `RetrieverConfig.tokenizer="code_cjk"`,
> not a feature flag — pure std, ADR-004/ADR-006). **ADR-029** is ratified `Proposed → Accepted` on the **real
> non-synthetic** verification of D1–D3/D5; the rust-native-eval-runner (D4) is **honestly deferred** per ADR-013.

### 3. 真实 tokenizer before/after recall delta（task-24.3，默认构建，真实）

`cargo run -p contextforge-core --example phase24_tokenizer_recall`（BM25 file-level over task-24.2 golden，default analyzer vs opt-in code/CJK analyzer，同一语料）：

| metric | before (default) | after (code/CJK) | delta |
|---|---|---|---|
| recall@5 / @10 / top-1 / MRR | 0.9091 | **1.0000** | **+0.0909** |

`corpus=9 expected + 3 distractor, queries=11`。delta 由真实 CJK case `语义检索` 驱动（default 整段 token miss，opt-in bigram `语义`/`检索` 命中 → after rank 1）；其余 10 query 两者 parity（full-symbol/full-phrase 在两 analyzer 均命中）。sub-token 判别力由 TEST-24.1.4 确定性背书。**小语料 caveat**（11 query / 12 file，承 task-19.5 §10）如实记录；无 query 退化。详 `docs/spikes/phase-24-tokenizer-recall.md`（ADR-013 真实非合成）。

## 5. Upgrade / Rollback

- Upgrade：默认构建行为不变（默认 tokenization + BM25 baseline + eval gate 阈值不变），无强制迁移。code/CJK tokenizer 经 `RetrieverConfig.tokenizer="code_cjk"` **opt-in**；**采用 opt-in 须 re-index 既有 collection**（opt-in 改 `content` 倒排词项，旧索引仍可用默认 analyzer 检索但不享受代码/CJK 子词命中）。eval 校验器 `ValidateGoldenSemantic` 为 add-only（既有 `ValidateDataset` 调用方不受影响）。
- Rollback：`git tag -d v0.17.0` + 删 GitHub Release / ghcr tag；默认镜像与 v0.16.0 行为兼容（默认 tokenization + BM25 + 0-dep），回退非破坏（opt-in tokenizer 未默认启用）。
