## 必含 entries（按 ADR-007）

| Entry                       | Mode   | 描述                                                                                            |
| --------------------------- | ------ | ----------------------------------------------------------------------------------------------- |
| `contextforge`              | 0o755  | Go control-plane binary (`cmd/contextforge`), executable bit required (`internal/release.ValidateTarball` enforces). |
| `contextforge-core`         | 0o755  | Rust data-plane binary (`cargo build -p contextforge-core`), executable bit required.            |
| `contextforge.example.toml` | 0o644  | Sample config — starting point for collection allowlists and local-only provider settings.       |
| `README.md`                 | 0o644  | Quick Start walkthrough (v0.2.0 rewrite — examples/quickstart/ fixture + flag order notes).     |
| `LICENSE`                   | 0o644  | All-rights-reserved placeholder until an OSI license is chosen.                                  |

`internal/release.ValidateTarball` rejects any tarball missing one of these entries or with `contextforge` lacking the executable bit (regression-guarded by `TestTask83_AC1_TarballContainsRequiredAssets`).

## v0.1.0 → v0.2.0 差异

- **proto schema**: 无 wire 变化（schema_version 仍 `0.1`，add-only `rpc Index` 不影响 binary compat）。
- **binary set**: 两个二进制（`contextforge` + `contextforge-core`），布局与 v0.1 完全一致。
- **README.md**: 重写 Quick Start 段，依赖 `examples/quickstart/` fixture（fixture 不打包到 tarball — 用户从 git checkout / 远端文档访问）。
- **scripts/release_smoke.sh**: 输出标记 `PHASE8_RELEASE_SMOKE_EXIT` → `PHASE_RELEASE_SMOKE_EXIT`；CI release job 需相应 update。

## Release 流程（外部 pipeline 责任）

本仓库提供：

- `scripts/release_smoke.sh`：可执行 release gate（4 段全 exit 0 才允许 tag）。
- `scripts/quickstart_smoke.sh`：用户体验回归 gate（7 step CLI 端到端真跑）。
- `internal/release.BuildTarball` + `internal/release.ValidateTarball`：编程式 tarball 构造 + 验证 API。

需要外部 release pipeline（GitHub Actions / 其它）执行：

- 真实 amd64 runner 上构建 `contextforge-linux-amd64-v0.2.0.tar.gz`。
- 计算并发布 sha256 checksum。
- `gh release create v0.2.0 ...` 上传 tarball + checksum + RELEASE_NOTES.md v0.2.0 section。
- 可选：GPG/codesign signing（v0.2 §3 OOS 沿 v0.1）。

这些 OOS 项保持 v0.1 task-8.3 §3 决策，不在 task-9.6 范围。
