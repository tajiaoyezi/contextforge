## Release 边界（继承 v0.1 + v0.2 + Phase 10 新增）

v0.3 在 v0.2 基础上新增：

- Console Contract v1 兼容层（v0.2 Console v1.0 ship 但 ContextForge 端无 REST 对接面 → v0.3 通过 ADR-015 真接通）
- 9 REST endpoint 严格对齐 Console HTTPAdapter（路径 / shape / 错误码 mapping）
- Workspace / IndexJob 资源模型（Rust 侧 SQLite 持久化）+ 异步 lifecycle + heartbeat + co-operative cancel
- ADR-014 cross-validation gate 首次完整激活（D1 + D2 + D3 + D4 + D5）

继承 v0.1 / v0.2 限制：

- 官方目标 Linux x86_64 / WSL2；macOS 应能跑（v0.3 smoke 测过），Windows 走 Git Bash / WSL。
- `LICENSE` 仍 all-rights-reserved。
- 真实 GitHub Release 上传、tag、checksum / signing、真实 amd64 runner 上的制品构建发布作业仍需外部 release pipeline 执行。

v0.3 特定限制（task-10.4/10.6 §10 trade-off）：

- Go REST 用 in-memory MemStore，跨进程 Rust↔Go SQLite 共享留 v0.4。
- Console UI docker image 公网未发布；console_smoke.sh 默认 local mode；DOCKER_SMOKE=1 + Console image env 三同时设才跑 full UI 集成。
- 其它 10+ Console endpoint（memory / eval / source-chunks / search trace / workspace config PATCH）留 v0.4。
