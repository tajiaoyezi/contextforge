# ContextForge v0.20.0 Evidence — memory-ops-hardening (pin-actor + pinned-at-timestamp + Pin/Unpin split + hard-delete + is_pinned audit backfill + ADR-032 ratified)

> v0.20.0 ships Phase 27: the Memory pin / lifecycle semantics landed in Phase 13 / Phase 17 are hardened,
> delivering the three ADR-022 §Trade-offs markers deliberately deferred at v0.10. **pin-actor +
> pinned-at-timestamp** become first-class add-only `MemoryItem` fields (audit log never recorded the calling
> actor) (task-27.1); **Pin/Unpin** is split explicitly (add-only `Unpin` RPC beside the `Pin{bool pin}`
> toggle) and a **hard-delete** strategy (physical row removal vs soft-delete's status flip, X-Confirm gated)
> is added (task-27.2); and **is_pinned** can be **backfilled from the audit log** (last `memory_pin`/
> `memory_unpin` event wins) (task-27.3). All proto changes are **add-only** (proto-freeze guard passes); the
> default build stays **0-new-dependency, 0-network** (ADR-004), and the existing 5 Memory RPC + `Pin` toggle
> + `confirmMiddleware` behavior do not regress. **ADR-032** is ratified `Proposed → Accepted` on the **real
> non-synthetic** verification of D1–D4; **ADR-022** gets an add-only Amendment recording the three markers.
