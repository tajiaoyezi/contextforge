## Upgrade path (v0.4.0 → v0.5.0)

1. **拉 v0.5.0 tag**：`git fetch --tags && git checkout v0.5.0`
2. **rebuild**：`cargo build -p contextforge-core --release && go build ./cmd/contextforge`
3. **重启 daemon**：旧 daemon 进程 SIGTERM；新 binary 启动会复用既有 data_dir
   (SqliteWorkspaceStore + SqliteJobStore + SqliteChunkStore schema 兼容 v0.4)
4. **Console UI 端**：BFF / Console Web 端如 strict 调用 cancel 200 → 切到 204
   handling (HTTPAdapter v1.0 已 200/204 双 check，应不需要改动)；PATCH config
   endpoint Console BFF 自动注入 X-Confirm: yes header (cross-repo PR 已 ship)；
   新 source-chunks / trace endpoint Console UI 已含端到端调用代码
5. **smoke 验证**：`bash scripts/console_smoke.sh` 期望 `CONSOLE_REAL_SMOKE_EXIT=0`
6. **ops curl 工作流**：调 PATCH config / memory deprecate / soft-delete 等
   destructive endpoint 须加 `-H 'X-Confirm: yes'` 或 `?confirm=true` query
