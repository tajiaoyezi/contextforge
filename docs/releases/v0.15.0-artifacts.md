# ContextForge v0.15.0 Artifacts — embedding-provider-completion (provider 配置选择 + 缓存 + 远程骨架 + ADR-027 ratified)

> Artifact inventory for the v0.15.0 minor release (Phase 22). Mirrors the v0.14.0 artifacts structure.
> The default build is unchanged, model-free, and 0-network-dep; the real embedding model
> (`embedding-fastembed`) and the remote provider (`embedding-remote`) are feature/deploy choices. This
> release completes the embedding **provider layer**: runtime `[embedding]` config selection + dim
> negotiation, a content-hash cache, a feature-gated remote HTTP skeleton, and an opt-in remote health
> probe — all add-only, no breaking contract bump (local-first stays the non-negotiable red line).

## 6. 版本声明（contract bump 评估）

- `[embedding]`(provider/dim) 为 **add-only** TOML 配置段（缺省 `Provider=""/Dim=0` → deterministic，向后兼容；既有 `[remote]`/`[[collections]]` 不受影响）。proto 零 delta（embedding provider 层为 core 库 + Go config，无 proto 字段变更）。Console Contract v1 shape 不变；22/31-step smoke 不破坏。
- 故 **无 breaking contract bump**；既有客户端无需改动。embedding 缓存为库装饰器（透明包裹）；远程 provider 经 `select_provider("remote")` + `embedding-remote` feature + 显式 opt-in（默认不激活）。

## 7. 后续模板（embedding-provider follow-on）
