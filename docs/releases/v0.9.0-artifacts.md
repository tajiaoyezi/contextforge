## 7. Trade-offs ship 矩阵 (Phase 16)

| Decision | Decision rationale | Cost / Future work |
|---|---|---|
| TraceStore write-through dual-write (task-16.1) | 内存 LRU 低延迟读 + SQLite SoT 持久化 | SQLite write 失败 swallow；不重试；缓解为 best-effort SoT |
| trace_json prost-encoded + base64 (task-16.1) | 与 PbRetrievalTrace prost-derive 一致 | 比 serde_json 体积小 ~30%；无 schema drift |
| SQLite 表无 LRU eviction (task-16.1) | 简单 INSERT OR REPLACE pattern | 长时间运行表可能数百万行；[SPEC-DEFER:phase-future.tracestore-sqlite-vacuum] |
| 两阶段 long-poll (task-16.2) | phase 1 block 等首 event + phase 2 短 drain 多 event | drainTimeout=100ms hard-coded；高并发场景可能 throughput 不够；缓解为 Console UI 自身节流 |
| MemStore fallback sleep(min(wait, 1s)) (task-16.2) | fallback 模式无真 event source 模拟阻塞 | 不能模拟真 event 触发；仅满足 contract 不破坏 — 与 fallback 模式整体限制一致 |
| linux/amd64 only image (task-16.3) | v0.9 简化；apple silicon 用户量不大 | arm64 build [SPEC-DEFER:phase-future.multi-arch-image] |
| 镜像不签名 / 无 SBOM (task-16.3) | v0.9 仅 build + push | cosign + SBOM [SPEC-DEFER:phase-future.image-signing-and-sbom] |
| CONTEXTFORGE_ALLOW_WILDCARD_BIND env opt-in (task-16.4) | ADR-004 安全基线 + 跨容器需 0.0.0.0 | 用户感知 + 显式 opt-in；不破坏默认 127.0.0.1 |
| compose-prod 不自动 letsencrypt TLS (task-16.4) | v0.9 假设外层 reverse proxy 处理 TLS | 自动 cert [SPEC-DEFER:phase-future.compose-tls-termination] |
| K8s Helm Chart 未提供 (task-16.4) | docker-compose 范例已足够 | Helm chart [SPEC-DEFER:phase-future.k8s-helm-chart] |
| Phase 17 / ADR-022 / task-17.1 单独 scaffold PR | v0.9 ship 本身不引新 ADR；Phase 17 await cross-repo | Pending → Ready 后再 ship task-17.1 实施 PR |
