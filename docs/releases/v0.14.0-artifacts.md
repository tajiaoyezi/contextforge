# ContextForge v0.14.0 Artifacts — retrieval-quality (hybrid scoring + reranker + ADR-025/026 ratified)

> Artifact inventory for the v0.14.0 minor release (Phase 21). Mirrors the v0.13.0 artifacts structure.
> The default build is unchanged and dependency-free; the real embedding model (`embedding-fastembed`)
> and real cross-encoder (`reranker-fastembed`) are feature/deploy choices. Retrieval quality this
> release adds **hybrid scoring** (RRF BM25+vector fusion) and a **reranker pipeline** (deterministic
> identity default + feature-gated real cross-encoder), both opt-in via add-only request fields / a
> builder seam — no breaking contract bump.

## 1. 二进制档

- `contextforge`（Go 控制面 CLI / console-api）：`eval run` 本 release 加 add-only `--hybrid`（`SearchRequest.Hybrid` → daemon `search_hybrid`）+ `--rerank`（eval 层确定性 IdentityReranker 重排 best base pass）flag，`SummarizePasses` 汇报 BM25 + semantic + hybrid + reranked 多路 recall；`--semantic` / 无 flag 时 byte-equivalent 既有输出。
- `contextforge-core`（Rust 数据面 daemon）：默认构建 = BM25 + 0-dep `BruteForceVectorBackend` 语义/hybrid 路径 + `DeterministicEmbeddingProvider` + 确定性 `IdentityReranker`（0 新 dep）；`server.rs` CoreService `req.hybrid` → `Retriever::search_hybrid`（RRF k=60 融合，task-21.1）；`Retriever::with_reranker` seam（task-21.2）。
- Feature 构建：`--features embedding-fastembed`（real `all-MiniLM-L6-v2` 向量分量）；`--features reranker-fastembed`（real cross-encoder `BGE-reranker-base`）；`--features vector-hnsw` 等 `vector-*` 见 ADR-023。

## 6. 版本声明（contract bump 评估）

- proto `SearchRequest.hybrid`(8)（task-21.1）+ `RetrievalResult.hybrid_score`(15) 为 **add-only**（`buf generate` 重生成 + proto-freeze 守护 PASS；Rust pb 经 `core/build.rs`）。Console Contract v1 shape 不变；22-endpoint conformance 不破坏；unset → BM25 行为。reranker 经 `Retriever::with_reranker` builder seam opt-in（无 proto 字段，默认 `None`）。
- 故 **无 breaking contract bump**；既有客户端无需改动。hybrid hits 经 `retrieval_method="hybrid"` + `hybrid_score`（融合分入既有 `score`）区分；reranked hits 经 `reason` provenance（`reranked:identity` / `reranked:cross-encoder`）标注。
