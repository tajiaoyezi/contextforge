> - **memory-actor-propagation**（兑现 ADR-032 §D1 `[SPEC-DEFER:phase-future.memory-actor-propagation]`）：`pin()` RPC 此前把调用 actor 硬编码 `"console-api"`——因 `PinMemoryRequest` 无 actor field、Go `MemoryClient.Pin(id,pin)` 无 actor 参数、`handleMemoryPin` 不读调用方标识；`set_pinned_with_actor` store 层自 task-27.1 已接受 actor，仅**入口到 store 的透传链缺失**。task-40.1 补：`PinMemoryRequest` add-only `string actor = 3`（既有 `memory_id=1` / `pin=2` 字段号冻结，ADR-015 D1）+ buf generate；Go `Pin(id,pin)` → `Pin(id,pin,actor)`（interface + `memoryClient` / `MemMemoryStore` / `degradedMemory` 三实现同步）+ `grpcclient` 填 `pb.PinMemoryRequest.Actor` + `handleMemoryPin` 读 `r.Header.Get("X-Actor")`（缺省空串）；Rust `pin()` `set_pinned_with_actor(.., if req.actor.is_empty() { "console-api" } else { req.actor.as_str() })`（空回落 byte-equiv）。console 部署在设 `X-Actor` / `X-Forwarded-User` 的 auth 代理后可把 pin/unpin 归因真实调用方（写入既有 `pinned_by`）。
> - **l2-cache-true-lru**（兑现 ADR-038 §A2/D4 `[SPEC-DEFER:phase-future.l2-cache-true-lru]`）：Phase 33 D1 给 L2 SQLite `embedding_cache` 加了 row-count cap + **rowid-FIFO**（插入序）驱逐，但 `sqlite_get` 命中**不重排** → 驱逐是插入序 FIFO 而非访问序 LRU。task-40.2 在 `sqlite_get` 命中分支（仅 `l2_cap > 0`）对命中行 `INSERT OR REPLACE` 原样回写以 bump 其隐式 rowid 到表尾 → 既有 `sqlite_put` 的 rowid 序驱逐随之由插入序 FIFO 升访问序 LRU。**复用既有隐式 rowid、0 schema migration**——**据实更正** Phase 33「真 LRU 须加 created_at 列 + ALTER 既有文件」的假设（命中 bump 即得，与 Go memstore 命中 move-to-front task-33.2 同技法）。
> - **ADR-045-ratified**：ADR-045 据 D1-D3 per-D ratify `Proposed → Accepted`；ADR-032（pin actor 透传维度兑现）/ ADR-038 + ADR-027（L2 true-LRU 维度兑现 + 真-LRU 假设据实更正）/ ADR-015（proto add-only field）add-only Phase-40 Amendment，**不**溯改各自 D-body（ADR-014 D5）。

### 2. l2-embedding-cache-true-lru（task-40.2，🟢）

`cargo test -p contextforge-core embedding::cache` PASS（9/9，含既有 TEST-33.1.* / 22.2.* 回归）：
- TEST-40.2.1（`test_40_2_1_l2_hit_bump_evicts_lru`）——`with_sqlite_capacity` cap=2：seed a,b → fresh-L1 provider `embed("a")` L2 命中（inner 不调，counter=0）→ 命中 bump a 的隐式 rowid 到表尾 → `embed("c")` miss（inner 调，counter=1）→ put 超 cap → 驱逐**最久未用** = b（`COUNT(*)`=2），a 存活；fresh-L1 provider 复读 a = L2 hit / b = L2 miss。对比 Phase 33 插入序 FIFO（命中不 bump）会驱逐 a——本测试断言区分 LRU vs FIFO。
- TEST-40.2.2（`test_40_2_2_cap_gates_hit_bump_and_keeps_results`）——(a) cap=2：seed a,b 后 `SELECT content_hash ORDER BY rowid` = `[key_a, key_b]`（插入序），fresh-L1 provider `embed("a")` 后 = `[key_b, key_a]`（命中 bump a 到表尾），返回 vector 不变（bump 原样回写相同字节）；(b) cap==0（不限）：seed a,b 后 fresh-L1 provider `embed("a")` 后 rowid 序仍 = `[key_a, key_b]`（**不** bump，保插入序、零写放大）。

**据实更正 Phase 33 假设（ADR-013）**：Phase 33（ADR-038 A2/D4）把真-LRU 据「带 created_at 列的真 LRU 须 ALTER 既有用户文件 → 破 0-migration」判定延后。本 phase grounding 更正——命中时对该行 `INSERT OR REPLACE`（`INSERT OR REPLACE` 对既有 PK 行先 DELETE 再 INSERT → 获新最大 rowid）即把隐式 rowid 由插入序变访问序，既有 `sqlite_put` 的 rowid 序驱逐随之升 LRU，**不须时间列、0 schema migration**，与 Go memstore 命中 move-to-front（task-33.2 / ADR-038 D2，同一代码库已有此范式）同技法。Phase 33 D1 的 rowid-FIFO（row-count cap 本身）是正确且必要的前序；本 phase 仅补「命中 bump 使 rowid 序由插入序变访问序」增量——以 add-only Amendment 记于 ADR-038/027，不溯改其正文（ADR-014 D5）。

### 3. honest-defer 边界（ADR-013 据实，MUST 记录）

**(a) pin actor 认证身份（不实现，另一层）**：本 phase 交付**调用方透传**——actor 取自 `X-Actor` header（调用方声明的标识），**未做认证校验**（把 header 值映射为已验证 auth subject 须 console-api 鉴权层，当前无）。据实记为「调用方声明的标识」非「已认证身份」→ honest-defer `[SPEC-DEFER:phase-future.memory-actor-authenticated-identity]`；其它 memory RPC 的 actor 透传续 `[SPEC-DEFER:phase-future.memory-actor-all-rpc]`。

**(b) L2 命中 bump 写放大 + opt-in-path 现网零影响**：命中 bump 给 L2 读路径加一次行重写（写放大），是访问序 LRU 的固有代价（同 Go memstore 命中 move-to-front 的内部代价）；且 `with_sqlite` 无生产调用点（Phase 33 D1 已据实标注 opt-in-path，出厂 daemon 走 memory-only L1）→ 现网零影响，本项是 **opt-in 路径语义补全、非已确认线上问题**（不夸大）。命中 bump 写放大优化续 `[SPEC-DEFER:phase-future.l2-lru-bump-batching]`；L2 TTL 续 `[SPEC-DEFER:phase-future.l2-cache-ttl]`；`with_sqlite` 生产接线续 `[SPEC-DEFER:phase-future.l2-cache-production-wire]`。

**(c) 其余治理 marker 据实保持延后（不强行扩面，honest over padding）**：`vector-dim-feature-enforce`（须 feature build：qdrant/lancedb/sqlite-vec 声明 collection dim）/ `tracestore-multi-workspace-strict`（余下读路径 / 内存路径深化）/ `chunk-source-type-filter` + `chunk-agent-scope-filter`（须 import-path schema migration + importer 侧标注）据 ADR-013 据实续延后，本焦点小版本不强行扩面。
