## tarball (manual build)

```bash
# Build both binaries
cargo build -p contextforge-core --release
go build -o contextforge ./cmd/contextforge

# Bundle
mkdir -p dist/contextforge-v0.4.0-linux-x86_64
cp target/release/contextforge-core dist/contextforge-v0.4.0-linux-x86_64/
cp contextforge dist/contextforge-v0.4.0-linux-x86_64/
cp config/example.toml dist/contextforge-v0.4.0-linux-x86_64/example.toml || true
cp README.md RELEASE_NOTES.md dist/contextforge-v0.4.0-linux-x86_64/
tar czf dist/contextforge-v0.4.0-linux-x86_64.tar.gz -C dist contextforge-v0.4.0-linux-x86_64
```

## 升级路径 (v0.3.0 → v0.4.0)

1. 拉 master @ `v0.4.0` tag.
2. 重新 build 两个 binary: `cargo build -p contextforge-core --release` + `go build ./cmd/contextforge`.
3. Deploy 双进程:
   ```bash
   # daemon (data plane gRPC)
   ./contextforge-core 127.0.0.1:50551 ~/.contextforge &
   # REST proxy (gRPC client)
   ./contextforge console-api-serve --addr 0.0.0.0:48181 --grpc-addr 127.0.0.1:50551 &
   ```
4. v0.3 用户若依赖 in-memory MemStore demo 行为: 设 `CONSOLE_API_FALLBACK_INMEM=1` (单进程模式; 不持久化跨重启).
5. v0.3 console_smoke.sh 用户: 设 `LOCAL_ONLY=1 bash scripts/console_smoke.sh` 仍跑 v0.3 inmem mode.
