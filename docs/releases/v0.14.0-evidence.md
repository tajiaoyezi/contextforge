# ContextForge v0.14.0 Evidence — retrieval-quality (hybrid scoring + reranker + ADR-025/026 ratified)

> v0.14.0 ships Phase 21: on top of the Phase 19/20 BM25 + semantic dual paths, it adds two opt-in
> ranking-quality enhancements — **hybrid scoring** (RRF fusion of the BM25 word-level + vector semantic
> scores, `retrieval_method = "hybrid"` + `hybrid_score`) and a **reranker pipeline** (`Reranker` trait +
> deterministic `IdentityReranker` default + feature-gated real cross-encoder `CrossEncoderReranker`).
> Both are opt-in: the **default build is unchanged and dependency-free** (still BM25 baseline, 0 new
> crate). **ADR-025** (hybrid fusion) and **ADR-026** (reranker provider) are ratified `Proposed →
> Accepted` on **real dogfood eval** data (ADR-013: no synthetic/fabricated figures). Default retrieval
> stays BM25; hybrid + rerank engage via explicit request.

## 3. 验证证据

### 1. 真实 hybrid / reranked recall vs BM25 baseline（task-21.3, real `FastEmbedProvider` `all-MiniLM-L6-v2` dim 384 + real `CrossEncoderReranker` BGE-reranker-base, 经生产 `Retriever::search` / `search_hybrid` / `with_reranker`, Windows MSVC, 2026-05-31）

`production_chunks=180 queries=30 backend=BruteForceVectorBackend(exact-cosine)`

| 检索法 | recall@5 | recall@10 | top-1 | MRR | ADR-006 gate (≥0.70) |
|---|---|---|---|---|---|
| baseline BM25 | 0.9000 | 0.9667 | 0.0333 | 0.4095 | **PASS** |
| **hybrid RRF**（ADR-025） | 0.9333 | 0.9667 | **0.6667** | **0.7881** | **PASS** |
| reranked cross-encoder（ADR-026） | **0.9667** | 0.9667 | 0.3333 | 0.6306 | **PASS** |

**ADR-025 ratify（hybrid）**：hybrid RRF 相对 BM25 baseline 取得决定性 top-1 +0.6334（0.0333→0.6667）/ MRR +0.3786（0.4095→0.7881）提升 → ADR-025 `Proposed→Accepted`，确认 D2 默认 RRF 策略（解 R1「合成数据不可区分」开放点，仿 ADR-023 数据驱动 ratify）。

**ADR-026 ratify（reranker）+ 诚实 caveat（ADR-013）**：real cross-encoder 在 Windows MSVC 成功跑通（D5 stop-condition 未触发），相对 BM25 baseline top-1 +0.30 / MRR +0.22 + 三法最高 recall@5（0.9667）→ ADR-026 `Proposed→Accepted`（架构/管道经真实模型端到端验证）。**诚实 caveat**：本小型 code+ADR 语料下，重排已强的 hybrid 融合 top-k **不及** hybrid 单路的 top-1（0.3333<0.6667）/ MRR（0.6306<0.7881）——通用 BGE 模型对代码块域适配弱 + 重排已优序只能动好序；rerank 真实价值在 recall@5（捞回漏召），故推荐为 **opt-in 域适配增强、非默认**（D4 opt-in seam 守默认构建从不重排）。数据源 + 方法 + ADR-013 三态声明见 `docs/spikes/phase-21-hybrid-recall.md`；复跑 `cargo run -p contextforge-core --example phase21_hybrid_rerank_recall --features embedding-fastembed,reranker-fastembed`。Offline note：HF 不可达时用 phase-20/phase-21-reranker spike 预取的本地 `.fastembed_cache` 模型，无伪造数值（ADR-013）。
