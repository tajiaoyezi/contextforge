`cargo test -p contextforge-core --features vector-qdrant retriever::vector::qdrant` **4 passed / 0 failed**（feature `vector-qdrant` 本机可构建——qdrant-client 1.18 携预生成 protobuf，无需 protoc）：

- TEST-25.1.3：collection ensure-create 决策 `decide_ensure`（喂入构造的 describe 响应，不连 server）— 存在且 dim/metric 匹配→`Reuse`、不存在→`Create`、存在但不匹配→`Error` 三分支。
- TEST-25.1.4：真实 KNN over live qdrant 诚实延后 `[SPEC-DEFER:phase-future.qdrant-server-lifecycle]`（CI 无 live server，禁伪造 live-server 通过）+ 生命周期方法为 inherent method 不破坏 task-18.1 三 trait 签名。
- **honest defer**：真实 connect→ensure-create→upsert→KNN over live qdrant server 因 CI 无在跑的 qdrant server 诚实延后（ADR-013：契约层 shape/config/decide_ensure 真实单测可断言；live-server 集成不伪造）。

### 2. lancedb 真实 dev-box 可构建性 🟢（task-25.2，feature `vector-lancedb`，真实构建 + 运行）

🟢 `cargo build --features vector-lancedb -p contextforge-core` **exit 0, 0 warnings**（`x86_64-pc-windows-msvc`, rustc 1.95.0, protoc `libprotoc 31.1` via 仓内 build-dep `protoc-bin-vendored` 3.2.0 经 `PROTOC` env）：
- 增量 feature 重建 13.78s；冷构建硬证据 = `target/debug/deps` **1097 依赖 rlib**（含 `liblancedb`≈144MB / `liblance`≈951MB / datafusion 53 + arrow 58 树）；registry src cache 构建前为空 → 构建后 50+ lance/datafusion/arrow crate 源（真实下载 + 编译，非合成）。
- **Cargo.lock 未变 → 0 新依赖**（lancedb 0.30 / arrow-array 58 / futures 0.3 自 task-18.5 即 optional）。
- `cargo test -p contextforge-core --features vector-lancedb --lib retriever::vector::lance_db` **2 passed / 0 failed**：TEST-25.2.4 真实 Lance dataset open→index→search KNN（查询 `[1,0,0,0]` 最近邻 `a`，`DistanceType::Cosine`）+ dim mismatch → `VectorError::DimMismatch`；TEST-25.2.3 `LanceIndexTuning::validate(dim)` 参数范围校验纯函数（partitions>0 / sub_vectors 整除 dim / m>0 / ef>0 / 阈值>0 / metric 受支持；非法 → 可识别 Err，不建真实索引）。
- **protoc 前置缩小非消除**：dev box PATH 内无 protoc / cmake，但仓内 `protoc-bin-vendored` 二进制经 `PROTOC` env 满足 lance `build.rs`（无需系统安装）→ 缩小 phase-18 protoc-prereq 担忧（仿 task-23.2 解除 sqlite-vec MSVC 受阻），但 protoc 仍须显式提供（不在 PATH / 未设 `PROTOC` 时 build.rs 失败）。**真实 ANN 索引建图 + 大语料性能** `[SPEC-DEFER:phase-future.lancedb-index-tuning]` / **compaction 执行** `[SPEC-DEFER:phase-future.lancedb-schema-compaction]` 诚实延后。详 `docs/spikes/phase-25-lancedb-buildability.md`（ADR-013 三态如实）。

### 3. 生产 backend 选择矩阵（task-25.3）

据 ADR-023 D1-D4 tier 排序 + ADR-028（Phase 23 嵌入式/fallback 推进）+ 本 phase qdrant/lancedb 推进结果，按「语料规模 × 部署形态 → 推荐 backend + caveat」收敛（矩阵为 add-only 指南，不溯改 ADR-023 D1-D6 tier 排序，D5）：

| 部署形态 / 语料规模 | 推荐 backend | tier 依据 | caveat |
|---|---|---|---|
| dev / 小语料 / 默认构建 | `BruteForceVectorBackend`（默认 0-dep）/ `hnsw` | ADR-023 D5 默认 0-vector-dep + D2 fallback | brute-force exact-cosine 适小语料；hnsw 重建图成本（ADR-028 rebuild-on-load） |
| 单机嵌入式持久 | `sqlite-vec`（feature `vector-sqlite`） | ADR-023 D1 嵌入式推荐默认 | 单机 MSVC 真实凭据（ADR-028，task-23.2 跨平台通过）；CI 默认不构建该 feature |
| 大语料嵌入式列存持久 | `lancedb`（feature `vector-lancedb`） | ADR-023 D4 embedded-columnar alternative | **protoc 前置**（须显式 `PROTOC`）+ 重 Arrow/DataFusion 栈（~5min 首构建）；🟢 Windows MSVC 可构建（task-25.2，单机凭据）；真实 ANN 索引性能延后 |
| hosted / multi-agent / scale-out | `qdrant`（feature `vector-qdrant`） | ADR-023 D3 hosted/scale-out | **需 live server**（`is_local()==false`，CI 无）；生命周期契约层已落地（task-25.1），真实 KNN over live server 延后 `[SPEC-DEFER:phase-future.qdrant-server-lifecycle]` |

矩阵口径：默认/小语料用 0-dep brute-force 或 hnsw fallback；规模/持久/部署形态升级时按上表选 feature-gated backend，每档 caveat 如实（live-server 依赖 / protoc 前置 / 平台限制 / 真实性能延后），ADR-013 不伪造未验证维度。
